Resumo do CodeIgniter¶

O CodeIgniter é um Framework de Aplicações¶

O CodeIgniter é um kit de ferramentas para quem constrói aplicações web usando PHP. Seu objetivo é permitir desenvolver projetos muito mais rápidos do que se você estivesse escrevendo código do zero, através de um abrangente conjunto de bibliotecas para as tarefas mais comuns, de uma simples interface e uma estrutura lógica para acessar essas bibliotecas. CodeIgniter permite que você mantenha o foco em seu projeto minimizando a quantidade de código necessário para uma determinada tarefa.

O CodeIgniter é Gratuíto¶

O CodeIgniter é disponibilizado sob licença MIT para que você possa usá-lo como desejar. Para mais informações, por favor leia a Licença de Uso.

O CodeIgniter é Leve¶

Realmente é muito leve. O núcleo do sistema requer apenas algumas pequenas bibliotecas. Isto está em contraste com muitos frameworks que requerem significativamente mais recursos. Bibliotecas adicionais são carregadas dinamicamente sob requisição, com base em suas necessidades de um determinado processo, de modo que o sistema básico é enxuto e bem rápido.

O CodeIgniter é Rápido¶

Realmente rápido. Te desafiamos a encontrar um framework com melhor performance que o CodeIgniter.

O CodeIgniter usa M-V-C¶

CodeIgniter usa a abordagem Model-View-Controller, que permite uma grande separação entre a lógica e a apresentação. Isto é particularmente bom para projetos em que os designers estão trabalhando com seus arquivos de template, como o código nesses arquivos serão o mínimo. Nós explicamos a abordagem MVC em mais detalhes em sua própria página.

O CodeIgniter gera URLs limpas¶

As URLs geradas pelo CodeIgniter são limpas e amigáveis aos motores de busca. Ao invés de usar a abordagem padrão "query string" para URLs que é sinônimo de sistemas dinâmicos, o CodeIgniter usa uma abordagem baseada em segmentos:

exemplo.com/noticias/artigo/345

Nota

Por padrão o arquivo index.php é incluído nas URL, mas pode ser removido usando um simples arquivo .htaccess.

O CodeIgniter vem recheado de coisas¶

CodeIgniter vem com uma grande variedade de bibliotecas que permitem que abrange a maioria das tarefas de desenvolvimento web, como o acesso a um banco de dados, envio de e-mail, validar os dados do formulário, mantendo sessões, manipulação de imagens, trabalhar com dados XML-RPC e muito mais.

O CodeIgniter é Extensível¶

O sistema pode ser facilmente expandido através do uso de suas próprias bibliotecas, helpers, ou através de extensões de classes ou ganchos de sistema.

O CodeIgniter não requer um Gerador de Templates¶

Embora o CodeIgniter venha com um simples analisador de templates que pode ser usado opcionalmente, ele não lhe forçá a usar um. Geradores de template simplesmente não pode igualar o desempenho do PHP nativo, e a sintaxe que deve ser aprendido para usá-los é geralmente um pouco mais fácil do que aprender o básico de PHP. Considere este bloco de código PHP:

<ul>
<?php foreach ($addressbook as $name):?>
        <li><?=$name?></li>
<?php endforeach; ?>
</ul>

Compare isso com o pseudo-código usado por um gerador de template:

<ul>
{foreach from=$addressbook item="name"}
        <li>{$name}</li>
{/foreach}
</ul>

Sim, segundo exemplo é um pouco mais claro, mas ele cobra seu preço na performance, pois um pseudo-código precisa ser convertido novamente em PHP pra rodar. Como um de nossos objetivos é máxima desempenho, optamos por não requerer o uso de um gerador de templates.

O CodeIgniter é Completamento Documentado¶

Programadores adoram codificar e odeiam documentar o que fazem. Nós não somos diferentes, é claro, mas como a documentação é tão importante quanto o código propriamente dito, estamos empenhados em fazê-lo. Nosso código fonte é extremamente limpo e também bem documentado.

O CodeIgniter tem uma comunidade de usuário amigável¶

Nossa crescente comunidade de usuários pode ser vista participando ativamento em nosso Fórum Comunitário.
