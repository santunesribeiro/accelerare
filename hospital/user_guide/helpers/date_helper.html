Auxiliar de Data¶

O arquivo Auxiliar de Data contém funções que auxiliam no trabalho com datas.

  • Carregando este Auxiliar
  • Funções disponíveis
  • Referência do fuso horário

Carregando este Auxiliar¶

Este auxiliar é carregado usando o seguinte código:

$this->load->helper('date');

Funções disponíveis¶

As seguinte funções estão disponíveis:

now([$timezone = NULL])¶
Parâmetros:
  • $timezone (string) – Fuso horário
Retorna:

Timestamp UNIX
Tipo de retorno:

int

Retorna a hora atual como um timestamp UNIX, referenciado à hora local do servidor ou a qualquer fuso horário suportado pelo PHP, com base na configuração "time reference (referência de tempo)" no seu arquivo de configuração. Se você não pretende definir sua referência de tempo principal para qualquer outro fuso horário suportado pelo PHP (o que normalmente você fará se você executar um site que permita a cada usuário definir suas próprias configurações de fuso horário), não há nenhum benefício para usar esta função em relação a função time() do PHP.

echo now('Australia/Victoria');

Se um fuso horário não for fornecido, ele retornará o time() com base na configuração time_reference.

mdate([$datestr = ''[, $time = '']])¶
Parâmetros:
  • $datestr (string) – String de data
  • $time (int) – Timestamp UNIX
Retorna:

Data formatada em MySQL
Tipo de retorno:

string

Esta função é idêntica à função date() do PHP, exceto que permite usar códigos de data no estilo MySQL, onde cada letra de código é precedida de um sinal de porcentagem, por exemplo, %Y %m %d

O benefício de fazer datas dessa maneira é que você não precisa se preocupar em escapar de quaisquer caracteres que não seja um código de data, como normalmente teria que fazer com a função date().

Exemplo:

$datestring = 'Year: %Y Month: %m Day: %d - %h:%i %a';
$time = time();
echo mdate($datestring, $time);

Se um timestamp não estiver incluído no segundo parâmetro, o tempo atual será usado.

standard_date([$fmt = 'DATE_RFC822'[, $time = NULL]])¶
Parâmetros:
  • $fmt (string) – Formato de data
  • $time (int) – Timestamp UNIX
Retorna:

Data formatada ou FALSE em caso de formato inválido
Tipo de retorno:

string

Permite gerar uma string de data em um dos vários formatos padronizados.

Exemplo:

$format = 'DATE_RFC822';
$time = time();
echo standard_date($format, $time);

Nota

Esta função está DEPRECIADA. Use a função nativa date() combinada com as constantes de formato do DateTime em vez disso:

echo date(DATE_RFC822, time());

Formatos suportados:

Constante Descrição Exemplo
DATE_ATOM Atom 2005-08-15T16:13:03+0000
DATE_COOKIE HTTP Cookies Sun, 14 Aug 2005 16:13:03 UTC
DATE_ISO8601 ISO-8601 2005-08-14T16:13:03+00:00
DATE_RFC822 RFC 822 Sun, 14 Aug 05 16:13:03 UTC
DATE_RFC850 RFC 850 Sunday, 14-Aug-05 16:13:03 UTC
DATE_RFC1036 RFC 1036 Sunday, 14-Aug-05 16:13:03 UTC
DATE_RFC1123 RFC 1123 Sun, 14 Aug 2005 16:13:03 UTC
DATE_RFC2822 RFC 2822 Sun, 14 Aug 2005 16:13:03 +0000
DATE_RSS RSS Sun, 14 Aug 2005 16:13:03 UTC
DATE_W3C W3C 2005-08-14T16:13:03+0000
local_to_gmt([$time = ''])¶
Parâmetros:
  • $time (int) – Timestamp UNIX
Retorna:

Timestamp UNIX
Tipo de retorno:

int

Toma um timestamp UNIX como entrada e retorna como GMT.

Exemplo:

$gmt = local_to_gmt(time());
gmt_to_local([$time = ''[, $timezone = 'UTC'[, $dst = FALSE]]])¶
Parâmetros:
  • $time (int) – Timestamp UNIX
  • $timezone (string) – Fuso horário
  • $dst (bool) – Se DST está ativo
Retorna:

Timestamp UNIX
Tipo de retorno:

int

Toma um timestamp UNIX (referenciado para GMT) como entrada e o converte em um timestamp localizado com base no fuso horário e no horário de verão enviado.

Exemplo:

$timestamp = 1140153693;
$timezone  = 'UM8';
$daylight_saving = TRUE;
echo gmt_to_local($timestamp, $timezone, $daylight_saving);

nota

Para uma lista de fuso horário, consulte a referência na parte inferior desta página.

mysql_to_unix([$time = ''])¶
Parâmetros:
  • $time (string) – MySQL timestamp
Retorna:

Timestamp UNIX
Tipo de retorno:

int

Toma um Timestamp do MySQL como entrada e o retorna como um timestamp UNIX.

Exemplo:

$unix = mysql_to_unix('20061124092345');
unix_to_human([$time = ''[, $seconds = FALSE[, $fmt = 'us']]])¶
Parâmetros:
  • $time (int) – Timestamp UNIX
  • $seconds (bool) – Se exibe os segundos
  • $fmt (string) – Formato (us ou euro)
Retorna:

Data formatada
Tipo de retorno:

string

Toma um timestamp UNIX como entrada e retorna em um formato legível para humanos com este protótipo:

YYYY-MM-DD HH:MM:SS AM/PM

Isso pode ser útil se você precisar exibir uma data em um campo de formulário para submissão.

O tempo pode ser formatado com ou sem segundos, e pode ser configurado para o formato europeu ou americano. Se apenas o timestamp for enviado, ele retornará o tempo sem segundos formatados para os EUA.

Exemplos:

$now = time();
echo unix_to_human($now); // Hora dos EUA, sem segundos
echo unix_to_human($now, TRUE, 'us'); // Hora dos EUA, com segundos
echo unix_to_human($now, TRUE, 'eu'); // Hora da Europa, com segundos
human_to_unix([$datestr = ''])¶
Parâmetros:
  • $datestr (int) – String de data
Retorna:

Timestamp UNIX ou FALSE em caso de falha
Tipo de retorno:

int

O oposto da função unix_to_time(). Toma um tempo "humano" como entrada e o retorna como um timestamp UNIX. Isso é útil se você aceitar as datas formatadas "humanas" enviadas por meio de um formulário. Retorna boolean FALSE caso uma string de data passada para ele não esteja formatado como indicado acima.

$now = time();
$human = unix_to_human($now);
$unix = human_to_unix($human);
nice_date([$bad_date = ''[, $format = FALSE]])¶
Parâmetros:
  • $bad_date (int) – Uma string de data terrivelmente formatada
  • $format (string) – Formato de data a retornar (igual à função date() do PHP)
Retorna:

Data formatada
Tipo de retorno:

string

Esta função pode ter um número de formatos de data mal formados e convertê-los em algo útil. Também aceita datas bem formadas.

A função retornará um timestamp UNIX por padrão. Você pode, opcionalmente, passar um formato em string (o mesmo tipo que a função date() do PHP aceita) como o segundo parâmetro.

Exemplo:

$bad_date = '199605';
// Deve Produzir: 1996-05-01
$better_date = nice_date($bad_date, 'Y-m-d');

$bad_date = '9-11-2001';
// Deve Produzir: 2001-09-11
$better_date = nice_date($bad_date, 'Y-m-d');
timespan([$seconds = 1[, $time = ''[, $units = '']]])¶
Parâmetros:
  • $seconds (int) – Número de segundos
  • $time (string) – Timestamp UNIX
  • $units (int) – Número de unidades de tempo para exibir
Retorna:

Diferença de tempo formatada
Tipo de retorno:

string

Formata um timestamp UNIX para que seja semelhante a este:

1 Year, 10 Months, 2 Weeks, 5 Days, 10 Hours, 16 Minutes

O primeiro parâmetro deve conter um timestamp UNIX. O segundo parâmetro deve conter um timestamp que seja maior que o primeiro timestamp. O terceiro parâmetro é opcional e limita o número de unidades de tempo a serem exibidas.

Se o segundo parâmetro estiver vazio, o tempo atual será usado.

O propósito mais comum para esta função é mostrar quanto tempo decorrido de algum momento no passado até agora.

Exemplo:

$post_date = '1079621429';
$now = time();
$units = 2;
echo timespan($post_date, $now, $units);

nota

O texto gerado por esta função é encontrado no seguinte arquivo de idioma: language/<your_lang>/date_lang.php

days_in_month([$month = 0[, $year = '']])¶
Parâmetros:
  • $month (int) – Um mês numérico
  • $year (int) – Um ano numérico
Retorna:

Contagem de dias no mês especificado
Tipo de retorno:

int

Retorna o número de dias em um determinado mês/ano. Tem em conta os anos bissextos.

Exemplo:

echo days_in_month(06, 2005);

Se o segundo parâmetro estiver vazio, o ano atual será usado.

nota

Esta função é um alias para o nativo cal_days_in_month(), se estiver disponível.

date_range([$unix_start = ''[, $mixed = ''[, $is_unix = TRUE[, $format = 'Y-m-d']]]])¶
Parâmetros:
  • $unix_start (int) – Timestamp UNIX da data de início do intervalo
  • $mixed (int) – Timestamp UNIX da data final do intervalo ou intervalo em dias
  • $is_unix (bool) – Configurado para FALSE se $mixed não for um timestamp
  • $format (string) – Formato da data de saída, igual ao date()
Retorna:

Um array de datas
Tipo de retorno:

array

Retorna uma lista de datas dentro de um período especificado.

Exemplo:

$range = date_range('2012-01-01', '2012-01-15');
echo "Primeiros 15 dias de 2012:";
foreach ($range as $date)
{
        echo $date."\n";
}
timezones([$tz = ''])¶
Parâmetros:
  • $tz (string) – Um fuso horário numérico
Retorna:

Diferença horária de UTC
Tipo de retorno:

int

Toma uma referência de fuso horário (para uma lista de fusos horários válidos, veja a "Referência do fuso horário" abaixo) e retorna o número de horas de deslocamento de UTC.

Exemplo:

echo timezones('UM5');

Esta função é útil quando usada com timezone_menu().

timezone_menu([$default = 'UTC'[, $class = ''[, $name = 'timezones'[, $attributes = '']]]])¶
Parâmetros:
  • $default (string) – Fuso horário
  • $class (string) – Nome da classe
  • $name (string) – Nome do menu
  • $attributes (mixed) – Atributos HTML
Retorna:

HTML drop down menu with time zones
Tipo de retorno:

string

Gera um menu suspenso de fuso horário, como este:

Este menu é útil se você executar um site de associação no qual seus usuários possam definir seu valor de fuso horário local.

O primeiro parâmetro permite definir o estado "selecionado" do menu. Por exemplo, para configurar o horário do Pacífico como o padrão você fará isso:

echo timezone_menu('UM8');

Consulte a referência do fuso horário abaixo para ver os valores deste menu.

O segundo parâmetro permite que você defina um nome de classe CSS para o menu.

O quarto parâmetro permite definir um ou mais atributos na tag de seleção gerada.

nota

O texto contido no menu é encontrado no seguinte arquivo de idioma: language/<your_lang>/date_lang.php
