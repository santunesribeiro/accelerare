Auxiliar de Arquivo¶

O arquivo Auxiliar de Arquivo contem funções que auxiliam no trabalho com arquivos.

  • Carregando este Auxiliar
  • Funções disponíveis

Carregando este Auxiliar¶

Este auxiliar é carregado usando o seguinte código:

$this->load->helper('file');

Funções disponíveis¶

As seguinte funções estão disponíveis:

read_file($file)¶
Parâmetros:
  • $file (string) – Caminho do arquivo
Retorna:

O conteúdo do arquivo ou FALSE em caso de falha
Tipo de retorno:

string

Retorna os dados contidos no arquivo especificado no caminho.

Exemplo:

$string = read_file('./caminho/para/arquivo.php');

O caminho pode ser um caminho de servidor relativo ou absoluto. Retorna FALSE (boleano) em caso de falha.

Nota

O caminho é relativo ao arquivo index.php principal no site, NÃO aos seus arquivos controller ou view. O CodeIgniter usa um front controller de modo que sempre será relativo ao index.php principal do site.

Nota

Esta função esta OBSOLETA. Use a função nativa file_get_contents() em seu lugar.

Importante

Se o seu servidor estiver executando uma restrição open_basedir esta função talvez não funcione se tentar acessar um arquivo que esteja um nível acima ao script chamado.

write_file($path, $data[, $mode = 'wb'])¶
Parâmetros:
  • $path (string) – Caminho do arquivo
  • $data (string) – Dados para gravar no arquivo
  • $mode (string) – modo fopen()
Retorna:

TRUE se a gravação foi bem sucedida, FALSE em caso de erro
Tipo de retorno:

bool

Escreve os dados no arquivo especificado no caminho. Se o arquivo não existir, ele será criado.

Exemplo:

$data = 'Alguns dados para o arquivo';
if ( ! write_file('./caminho/para/arquivo.php', $data))
{
        echo 'Não foi possível escrever no arquivo';
}
else
{
        echo 'Dados escritos!';
}

Você pode configurar opcionalmente o modo de escrita através do terceiro parâmetro:

write_file('./caminho/para/arquivo.php', $data, 'r+');

O modo padrão é ‘wb’. Por favor, veja o Guia do Usuário do PHP para mais opções.

Nota

O caminho é relativo ao arquivo index.php principal no site, NÃO aos seus arquivos controller ou view. O CodeIgniter usa um front controller de modo que sempre será relativo ao index.php principal do site.

Nota

Esta função requer um bloqueio exclusivo no arquivo ao escrever nele.

delete_files($path[, $del_dir = FALSE[, $htdocs = FALSE]])¶
Parâmetros:
  • $path (string) – Caminho do diretório
  • $del_dir (bool) – Se deseja também excluir diretórios
  • $htdocs (bool) – Se deseja ignorar a exclusão de arquivos da página .htaccess e index
Retorna:

TRUE no sucesso, FALSE em caso de um erro
Tipo de retorno:

bool

Apaga TODOS os arquivos dentro do caminho fornecido.

Exemplo:

delete_files('./caminho/para/diretório/');

Se o segundo parâmetro estiver configurado como TRUE, qualquer diretório dentro do caminho fornecido também será apagado.

Exemplo:

delete_files('./caminho/para/diretório/', TRUE);

Nota

Os arquivos devem ter permissão de escrita ou pertencerem ao sistema de modo que possam ser apagados.

get_filenames($source_dir[, $include_path = FALSE])¶
Parâmetros:
  • $source_dir (string) – Caminho do diretório
  • $include_path (bool) – Se deseja incluir o caminho como parte dos nomes dos arquivos
Retorna:

Um array dos nomes dos arquivos
Tipo de retorno:

array

Toma um caminho do servidor como entrada e retorna um array contendo os nomes de todos arquivos contidos nele. O caminho do arquivo pode opcionalmente ser adicionado aos nomes dos arquivos, definindo o segundo parâmetro como TRUE.

Exemplo:

$controllers = get_filenames(APPPATH.'controllers/');
get_dir_file_info($source_dir, $top_level_only)¶
Parâmetros:
  • $source_dir (string) – Caminho do diretório
  • $top_level_only (bool) – Se deseja observar apenas o diretório especificado (excluindo subdirectórios)
Retorna:

Um array contendo informações sobre o conteúdo do diretório fornecido
Tipo de retorno:

array

Lê um diretório especificado e constrói um array contendo os nomes dos arquivos, tamanho dos arquivos, datas e permissões. Os subdiretórios dentro do caminho especificado é lido somente se forçado passando o segundo parâmetro como FALSE, pois isso pode ser uma operação intensiva.

Exemplo:

$models_info = get_dir_file_info(APPPATH.'models/');
get_file_info($file[, $returned_values = array('name', 'server_path', 'size', 'date')])¶
Parâmetros:
  • $file (string) – Caminho do arquivo
  • $returned_values (array) – Qual o tipo da informação a retornar
Retorna:

Um array contendo informações sobre o arquivo especificado ou FALSE em caso de falha
Tipo de retorno:

array

Com base em um arquivo e um caminho, retorna (opcionalmente) informações de atributos como nome, caminho, tamanho e data de modificação para um arquivo. O segundo parâmetro permite que você declare explicitamente qual informação deseja que retorne.

As opções válidas em $returned_values são: name, size, date, readable, writeable, executable e fileperms.

get_mime_by_extension($filename)¶
Parâmetros:
  • $filename (string) – Nome do arquivo
Retorna:

MIME tipo string ou FALSE em caso de falha
Tipo de retorno:

string

Traduz uma extensão de um nome de arquivo em um tipo MIME baseado em config/mimes.php. Retorna FALSE se não pode determinar o tipo ou ler o arquivo de configuração MIME.

$file = 'somefile.png';
echo $file.' is has a mime type of '.get_mime_by_extension($file);

Nota

Esta não é uma maneira exata de determinar os tipos MIME de arquivos, e está aqui estritamente por conveniência. Não deve ser usada para fins de segurança.

symbolic_permissions($perms)¶
Parâmetros:
  • $perms (int) – Permissões
Retorna:

String de permissões simbólicas
Tipo de retorno:

string

Assume permissões numéricas (como é retornado pelo fileperms()) e retorna notação simbólica padrão de permissões de arquivo.

echo symbolic_permissions(fileperms('./index.php'));  // -rw-r--r--
octal_permissions($perms)¶
Parâmetros:
  • $perms (int) – Permissões
Retorna:

String de permissões Octal
Tipo de retorno:

string

Assume permissões numéricas (como é retornado pelo fileperms()) e retorna uma notação octal de três caracteres de permissões de arquivo.

echo octal_permissions(fileperms('./index.php')); // 644
