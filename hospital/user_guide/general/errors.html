Tratamento de Erros¶

O CodeIgniter permite que você crie relatórios de erros em seus aplicativos usando as funções descritas abaixo. Além disso, ele possui uma classe de registro de erros que permite que mensagens de erro e depuração sejam salvas como arquivos de texto.

Nota

Por padrão, o CodeIgniter exibe todos os erros do PHP. Você pode querer mudar esse comportamento assim que seu desenvolvimento for concluído. Você encontrará a função error_reporting() localizada na parte superior do arquivo index.php principal. Desativar o relatório de erros NÃO impedirá que os arquivos de log sejam escritos se houver erros..

Ao contrário da maioria dos sistemas, no CodeIgniter, as funções de erro são simples interfaces processuais que estão disponíveis globalmente ao longo do aplicativo. Essa abordagem permite que as mensagens de erro sejam ativadas sem ter que se preocupar com o escopo da classe/função.

CodeIgniter também retorna um código de status sempre que uma parte do núcleo chama exit(). Este código de status de saída é separado do código de status HTTP e serve como um aviso para outros processos que podem estar observando se o script foi concluído com sucesso ou, caso contrário, qual o tipo de problema que ele encontrou que causou o aborto Esses valores são definidos em application/config/constants.php. Enquanto os códigos de status de saída são mais úteis nas configurações de CLI, retornar o código apropriado ajuda o software do servidor a acompanhar seus scripts e a saúde de sua aplicação.

As seguintes funções permitem gerar erros:

show_error($message, $status_code, $heading = 'An Error Was Encountered')¶
Parâmetros:
  • $message (mixed) – Mensagem de erro
  • $status_code (int) – Código de status HTTP de resposta
  • $heading (string) – Cabeçalho da página de erro
Tipo de retorno:

nada

Esta função exibirá a mensagem de erro fornecida com o modelo de erro apropriado para sua execução:

application/views/errors/html/error_general.php

ou:

application/views/errors/cli/error_general.php

O parâmetro opcional $status_code determina qual código de status HTTP deve ser enviado com o erro. Se $status_code for inferior a 100, o código de status HTTP será definido como 500 e o código de status de saída será definido como $status_code + EXIT__AUTO_MIN. e esse valor for maior do que EXIT__AUTO_MAX, ou se $status_code for 100 ou superior, o código de status de saída será configurado para EXIT_ERROR. Você pode verificar em application/config/constants.php para mais detalhes.

show_404($page = '', $log_error = TRUE)¶
Parâmetros:
  • $page (string) – Conjunto de caracteres da URI
  • $log_error (bool) – Se deseja registrar o erro
Tipo de retorno:

nada

Esta função exibirá a mensagem de erro 404 fornecida com o modelo de erro apropriado para sua execução:

application/views/errors/html/error_404.php

ou:

application/views/errors/cli/error_404.php

A função espera que o conjunto de caracteres passado para ele, seja o caminho do arquivo para a página que não foi encontrada. O código de status de saída será configurado para EXIT_UNKNOWN_FILE. Observe que CodeIgniter mostra automaticamente as mensagens de 404 se os controladores não forem encontrados.

O CodeIgniter registra automaticamente todas as chamadas show_404(). A configuração do segundo parâmetro opcional para FALSE irá ignorar o registro.

log_message($level, $message)¶
Parâmetros:
  • $level (string) – Nível de registro: ‘error’, ‘debug’ or ‘info’
  • $message (string) – Menssagem para o registro
Tipo de retorno:

nada

Esta função permite que você escreva mensagens para seus arquivos de log. Você deve fornecer um dos três "níveis" no primeiro parâmetro, indicando o tipo de mensagem que é (erro,depurar, informações), com a própria mensagem no segundo parâmetro.

Exemplo:

if ($some_var == '')
{
        log_message('error', 'Alguma variável não contém um valor.');
}
else
{
        log_message('debug', 'Alguma variável foi configurada corretamente');
}

log_message('info', 'O objetivo de alguma variável é fornecer algum valor.');

Existem três tipos de mensagens:

  1. Mensagens de erro. Estes são erros reais, como erros de PHP ou erros do usuário.
  2. Mensagens de depuração. Estas são mensagens que ajudam na depuração. Por exemplo, se uma classe foi inicializada, você pode registrar isso como informações de depuração.
  3. Mensagens Informativas. Estas são as mensagens de menor prioridade, simplesmente fornecendo informações sobre algum processo.

Nota

Para que o arquivo de log seja realmente escrito o diretório logs/ deve ser gravável. Além disso, você deve definir o “limite” para iniciar sessão em application/config/config.php. Você pode, por exemplo, apenas querer que as mensagens de erro sejam registradas e não os outros dois tipos. Se você configurá-lo para zero, o registro será desativado
