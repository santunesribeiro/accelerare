Perfis de sua Aplicação¶

A classe Perfis de Aplicação exibirá resultados de pontos de referência, consultas que você executou $_POST na parte inferior das suas páginas. Esta informação pode ser útil durante o desenvolvimento, afim de ajudar na depuração e otimização.

Inicializando a Classe¶

Importante

Esta classe NÃO precisa ser inicializada. Ele é carregado automaticamente pela Biblioteca de Saída se o perfil for habilitado como mostrado abaixo.

Habilitando os Perfis¶

Para habilitar o perfil, coloque a seguinte linha em qualquer lugar dentro dos métodos do Controlador:

$this->output->enable_profiler(TRUE);

Quando ativado, um relatório será gerado e inserido na parte inferior de suas páginas.

Para desativar o perfil, você usará:

$this->output->enable_profiler(FALSE);

Definindo Pontos de Referência¶

Para que o Perfil compile e exiba seus dados de referência, você deve nomear seus pontos de marca usando a sintaxe específica.

Leia as informações sobre a configuração dos pontos de referência na página da Biblioteca de Referência.

Habilitando e Desabilitando Seções dos Perfis¶

Cada seção de dados do Perfil pode ser ativada ou desativada definindo uma variável de configuração correspondente como VERDADEIRA ou FALSA. Isso pode ser feito de duas formas diferentes. Primeiro, você pode definir padrões de aplicativos com o arquivo de configuração application/config/profiler.php.

Exemplo:

$config['config']          = FALSE;
$config['queries']         = FALSE;

Em seus controladores, você pode substituir os padrões e os valores dos arquivos de configuração, chamando o método set_profiler_sections() da Biblioteca de Saída:

$sections = array(
        'config'  => TRUE,
        'queries' => TRUE
);

$this->output->set_profiler_sections($sections);

As seções disponíveis e a chave de matriz usada para acessá-las estão descritas na tabela abaixo.

Chave Descrição Padrão
benchmarks Tempo decorrido dos pontos de referência e o tempo total de execução TRUE
config Variáveis de configuração do Codeigniter TRUE
controller_info A classe e método do controlador solicitado TRUE
get Qualquer dado passado com GET na requisição TRUE
http_headers Os cabeçalhos HTTP para a requisição atual TRUE
memory_usage Quantidade de memória consumida pela requisição atual, em bytes TRUE
post Qualquer dado passado com POST na requisição TRUE
queries Listagem de todas as consultas de banco de dados executadas, incluindo o tempo de execução TRUE
uri_string A URI da requisição atual TRUE
session_data Dados armazenados na sessão atual TRUE
query_toggle_count O número de consultas após o qual o bloco de consulta será o padrão para oculto. 25

Nota

Desativar a configuração save_queries na configuração do banco de dados também irá efetivamente desabilitar o perfil para consultas do banco de dados e tornar a configuração de "consultas" acima inútil. Você pode, opcionalmente, anular esta configuração com $this->db->save_queries = TRUE;. Sem essa configuração, você não poderá visualizar as consultas ou a last_query <database/helpers>.
