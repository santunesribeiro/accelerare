Controladores¶

Os controladores são o coração do seu aplicativo, pois determinam como os pedidos HTTP devem ser tratados.

O que é um Controlador?¶

Um controlador é simplesmente um arquivo de classe que é nomeado de uma maneira que pode ser associada a um URI.

Considerando esta URI:

example.com/index.php/blog/

No exemplo acima, o CodeIgniter tentaria encontrar um controlador chamado Blog.php e carregá-lo.

Quando o nome de um controlador corresponder ao primeiro segmento de uma URI, ele será carregado.

Vamos tentar: Olá Mundo!¶

Vamos criar um controlador simples para que você possa vê-lo em ação. Usando seu editor de texto, crie um arquivo chamado Blog.php e coloque o seguinte código nele:

<?php
class Blog extends CI_Controller {

        public function index()
        {
                echo 'Olá Mundo!';
        }
}

Em seguida salve o arquivo no diretório application/controllers/.

Importante

O arquivo deve ser chamado ‘Blog.php’, com um ‘B’ maíusculo.

Agora visite o seu site usando um URL semelhante a esta:

example.com/index.php/blog/

Se você fez certo, você deverá ver:

Olá Mundo!

Importante

Os nomes das classes devem começar com uma letra maiúscula.

Isso é válido:

<?php
class Blog extends CI_Controller {

}

Isso não é válido:

<?php
class blog extends CI_Controller {

}

Além disso, certifique-se sempre de que o seu controlador estende a classe do controlador principal para que ele possa herdar todos os seus métodos.

Métodos¶

No exemplo acima, na criação do controlador, o nome do método é index(). O método “index” será carregado sempre por padrão se o segundo segmento da URI estiver vazio. Outra maneira de mostrar sua mensagem “Olá Mundo” seria assim:

example.com/index.php/blog/index/

O segundo segmento do URI determina qual método no controlador é chamado.

Vamos tentar. Adicione um novo método ao seu controlador:

<?php
class Blog extends CI_Controller {

        public function index()
        {
                echo 'Olá Mundo!';
        }

        public function comments()
        {
                echo 'Olhe isso!';
        }
}

Agora carregue a seguinte URL para ver o método de comentário:

example.com/index.php/blog/comments/

Você deve ver sua nova mensagem.

Passando Segmentos da URI para seus Métodos¶

Se a sua URI contiver mais de dois segmentos, eles serão passados ​​para o seu método como parâmetros.

Por exemplo, digamos que você tenha uma URI como esta:

example.com/index.php/products/shoes/sandals/123

Seu método de URI irá passar os segmentos 3 e 4 (“sandals” e “123”):

<?php
class Products extends CI_Controller {

        public function shoes($sandals, $id)
        {
                echo $sandals;
                echo $id;
        }
}

Importante

Se você está usando características de Roteamento URI , Os segmentos passados ​​para o seu método serão os reencaminhados.

Definindo o Controlador Padrão¶

O CodeIgniter pode ser informado para carregar um controlador padrão quando uma URI não está presente, como será o caso quando apenas o URL de raiz do seu site for solicitado. Para especificar um controlador padrão, abra o arquivo emapplication/config/routes.php e defina esta variável:

$route['default_controller'] = 'blog';

Onde ‘blog’ é o nome da classe de controlador que você deseja usar. Se você carregar seu arquivo index.php principal sem especificar nenhum segmento URI, você verá sua mensagem “Olá Mundo” por padrão.

Para obter mais informações, consulte o manual para “Rotas Reservadas” na seção Roteamento URI.

Remapeando Chamadas de Métodos¶

Conforme observado acima, o segundo segmento da URI tipicamente determina qual método no controlador é chamado. O CodeIgniter permite que você substitua esse comportamento através do uso do método_remap():

public function _remap()
{
        // Alguma coisa aqui...
}

Importante

Se o seu controlador contiver um método chamado _remap(), este sempre será chamado independentemente do que a sua URI contenha. Ele substitui o comportamento normal no qual a URI determina qual método é chamado, permitindo que você defina suas próprias regras de roteamento de métodos.

A chamada do método substituído (normalmente o segundo segmento do URI) será passada como um parâmetro para o método _remap():

public function _remap($method)
{
        if ($method === 'algum_metodo')
        {
                $this->$method();
        }
        else
        {
                $this->default_method();
        }
}

Quaisquer segmentos extras após o nome do método são passados ​​para _remap() Como um segundo parâmetro opcional. Esta matriz pode ser usada em combinação com o método PHP call_user_func_array() para emular o comportamento padrão do CodeIgniter..

Exemplo:

public function _remap($method, $params = array())
{
        $method = 'process_'.$method;
        if (method_exists($this, $method))
        {
                return call_user_func_array(array($this, $method), $params);
        }
        show_404();
}

Processamento de Saída¶

O CodeIgniter tem uma classe de saída que cuida enviar seus dados renderizados finais para o navegador da Web automaticamente. Mais informações sobre isso podem ser encontradas nas páginas Views e Classe de Saída. Em alguns casos, no entanto, você pode querer pós-processar os dados finalizados de alguma forma e enviá-lo para o navegador você mesmo. O CodeIgniter permite que você adicione um método chamado _output () ao seu controlador que receberá os dados de saída finalizados. _output()

Important

Se o seu controlador contiver um método chamado _output(), este sempre será chamado pela classe de saída em vez de fazer eco dos dados finalizados diretamente. O primeiro parâmetro do método conterá a saída finalizada.

Aqui está um exemplo:

public function _output($output)
{
        echo $output;
}

Nota

Por favor, note que seu método _output() receberá os dados em seu estado finalizado. Os dados de referência e de uso de memória serão renderizados, os arquivos de cache escritos (se você tiver habilitado o armazenamento em cache) e os cabeçalhos serão enviados (se você usar esse recurso) antes de ser transferido para o método _output(). Para que a saída do seu controlador seja armazenada em cache corretamente, o método _output() pode usar:

if ($this->output->cache_expiration > 0)
{
        $this->output->_write_cache($output);
}

Se você estiver usando este recurso, o tempo de execução da página e as estatísticas de uso da memória podem não ser perfeitamente precisas, pois não levarão em consideração qualquer processamento posterior que você faça. Por uma maneira alternativa de controlar a saída antes de qualquer um dos processos finais, ver os métodos disponíveis em Biblioteca de Saída.

Métodos Privados¶

Em alguns casos, você pode querer certos métodos escondidos do acesso público. Para conseguir isso, simplesmente declare o método como privado ou protegido e não será servido por meio de uma solicitação de URL. Por exemplo, se você tivesse um método como este:

private function _utility()
{
        // algum código
}

Tentando acessá-lo através do URL, assim, não funcionará:

example.com/index.php/blog/_utility/

Nota

Os nomes dos métodos de prefixação com um título de sublinhado também impedirão que sejam chamados. Este é um recurso herdado que é deixado para compatibilidade com versões anteriores.

Organizando seus Controladores em Sub-diretórios¶

Se você estiver construindo um aplicativo grande, você pode querer hierarquicamente organizar ou estruturar seus controladores em subdiretórios. O CodeIgniter permite que você faça isso..

Simplesmente crie subdiretórios sob o principal application/controllers/ e coloque suas classes de controle dentro deles.

Nota

Ao usar esse recurso, o primeiro segmento do URI deve especificar a pasta. Por exemplo, digamos que você tenha um controlador localizado aqui:

application/controllers/products/Shoes.php

Para chamar o controlador acima, a sua URI será algo como isto:

example.com/index.php/products/shoes/show/123

Cada um dos seus subdiretórios pode conter um controlador padrão que será chamado se o URL contiver apenas o subdiretório. Basta colocar um controlador lá que combine o nome do seu ‘controlador_padrao’ especificado em seu arquivo application/config/routes.php.

CodeIgniter também permite que você remapeie seus URIs usando o recurso Rotas de URI .

Construtores de classe¶

Se você pretende usar um construtor em qualquer um de seus Controladores, você DEVE inserir a seguinte linha de código:

parent::__construct();

A razão pela qual esta linha é necessária é porque seu construtor local irá substituir o da classe do controlador principal, então precisamos chamá-lo manualmente.

Exemplo:

<?php
class Blog extends CI_Controller {

        public function __construct()
        {
                parent::__construct();
                // Seu próprio código de construtor
        }
}

Os Construtores são úteis se você precisar definir alguns valores padrão ou executar um processo padrão quando sua classe for instanciada. Os Construtores não podem retornar um valor, mas eles podem fazer algum trabalho padrão.

Nomes de Métodos Reservados¶

Uma vez que suas classes de controle estenderão o controlador de aplicativo principal, você deve ter cuidado para não nomear seus métodos de forma idêntica aos usados ​​por essa classe, caso contrário, suas funções locais os substituirão. Veja a lista completa em Nomes Reservados.

Importante

Você também não deve ter um método denominado de forma idêntica ao nome da classe. Se você fizer isso, e não há nenhum método __construct() Método na mesma classe, então ex: O método Index::index() será executado como um construtor de classe! Este é um recurso de compatibilidade com o PHP4

É isso!¶

Isso, em poucas palavras, é tudo o que há para saber sobre os controladores.
