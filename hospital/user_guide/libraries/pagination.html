Classe de Paginação¶

A classe de paginação do CodeIgniter é muito simples de ser utilizada e 100% personalizável (dinamicamente ou através configurações armazenadas).

Se você não está familiarizado com o termo “paginação”, isso se refere aos links de navegação, que permitem a visualização página a página, assim como estes: 

« Primeira  < 1 2 3 4 5 >  Última »

Exemplo¶

Aqui está um simples exemplo demonstrando como criar paginação em um de seus métodos do controller:

$this->load->library('pagination');

$config['base_url'] = 'http://example.com/index.php/test/page/';
$config['total_rows'] = 200;
$config['per_page'] = 20;

$this->pagination->initialize($config);

echo $this->pagination->create_links();

Notas¶

O array $config contém suas variáveis de configuração. Ela é passada ao método$this->pagination->initialize() como é mostrado acima. Embora exista cerca de 20 itens que você pode configurar, no mínimo 3 precisam ser mostrados. Aqui está a descrição do que estes itens representam:

  • base_url Este é URL completo da classe controller/função que contém sua paginação. No exemplo acima, está apontando para o controller chamado “Test” e uma função chamada “page”. Lembre-se que você pode reorganizar seu roteamento URI se você necessita de uma estrutura diferente.
  • total_rows Este número representa o total de linhas do resultado o qual você está criando a paginação. Geralmente este número será o total de linhas retornadas pela pesquisa no banco.
  • per_page O número de itens que você pretende mostrar por página. No exemplo acima, seria exibido 20 itens por página.

O método create_links() retorna uma string vazia quando não há paginação a ser exibida.

Definindo as preferências em um arquivo de paginação¶

Se você prefere definir as preferências utilizando o método acima, você pode colocar as definições em um arquivo de configurações. Apenas crie um novo arquivo chamado pagination.php, adicione o array $config dentro do arquivo criado, salve o arquivo na pasta: application/config/ e ele será utilizado automaticamente. Você não precisa usar $this->pagination->initialize() se você salvar suas preferências no arquivo de configurações.

Personalizando a Paginação¶

A seguir está a lista com todas as preferências que você poderá passar ao método de inicialização para adaptar na tela.

$config[‘uri_segment’] = 3;

A função de paginação automaticamente determina que segmento da sua URI contém o número da paǵina. Se necessita que seja um segmento diferente, você pode especificar:

$config[‘num_links’] = 2;

O número de links de número de páginas que você gostaria de exibir antes e depois do número da página selecionada. Por exemplo, o número 2 colocará dois dígitos (links de páginas) de cada lado do link da página atual, como o exemplo exibido no topo da página.

$config[‘use_page_numbers’] = TRUE;

Por padrão, o segmento da URI utilizará o índice inicial para os itens que você está paginando. Se você prefere exibir o número da página atual, defina o valor como "TRUE".

$config[‘page_query_string’] = TRUE;

Por padrão, a biblioteca de paginação assume que você está utilizando URI Segments, e constrói seus links como:

http://example.com/index.php/test/page/20

Se você possui $config['enable_query_strings'] definido como TRUE seus links serão automaticamente reescritos utilizando Query Strings. Esta opção também pode ser definida explicitamente. Usando $config['page_query_string'] definido como TRUE, o link de paginação se tornará: 

http://example.com/index.php?c=test&m=page&per_page=20

Note que “per_page” é a requisição passada como padrão, entretanto pode ser configurada utilizando $config['query_string_segment'] = 'sua_string'

$config[‘reuse_query_string’] = FALSE;

Por padrão seus argumentos de requisição (nada a ver com outros opções de string de consulta) serão ignorados. Definindo esta configuração para TRUE, serão adicionados strings de busca existentes novamente à URL depois do segmento de URI e antes do sufixo:

http://example.com/index.php/test/page/20?query=search%term

Isto ajuda a juntar o URI Segments normal assim como argumentos de requisição (parâmetros de consulta), que até a versão 3.0 no era possível.

$config[‘prefix’] = ‘’;

Um prefixo personalizado ao endereço. O valor do prefixo estará antes do segmento offset.

$config[‘suffix’] = ‘’;

Um sufixo adicionado ao endereço. O valor do sufixo estará depois do segmento offset.

$config[‘use_global_url_suffix’] = FALSE;

Quando definido como TRUE, Serásobreposto o valor de $config['suffix'] e em vez de definir com o que está em $config['url_suffix'] no seu arquivo application/config/config.php.

Adicionando Marcações (tag html) ¶

Se você gostaria de cercar a paginação com algumas marcações (tags html), você pode fazê-lo com estas duas preferências:

$config[‘full_tag_open’] = ‘<p>’;

A tag de abertura do lado esquerdo do resultado total.

$config[‘full_tag_close’] = ‘</p>’;

A tag de fechamento localizada no lado esquerdo do resultado total.

Personalizando o Primeiro Link ¶

$config[‘first_link’] = ‘Primeira’;

O texto a ser exibido no “primeiro” link à esquerda. Se prefere não renderizar este link, você pode definir o valor como FALSO.

Nota

Este valor também pode ser traduzido via arquivo de linguagem.

$config[‘first_tag_open’] = ‘<div>’;

A tag de abertura para o “primeiro” link.

$config[‘first_tag_close’] = ‘</div>’;

A tag de fechamento para o “primeiro” link.

$config[‘first_url’] = ‘’;

Uma URL alternativa para utilizar para o link de “primeira página”.

Personalizando o link Último¶

$config[‘last_link’] = ‘Última’;

O texto que será exibido no link “último” na direita. Para não ter esse link renderizado, pode ser definido este valor para FALSE.

Nota

Este valor também pode ser traduzido via arquivo de linguagem.

$config[‘last_tag_open’] = ‘<div>’;

A tag de abertura para o link “último”.

$config[‘last_tag_close’] = ‘</div>’;

A tag de fechamento para o link “último”.

Personalizando o link “Próximo”¶

$config[‘next_link’] = ‘&gt;’;

O texto que será exibido no link da página “Próxima”. Se não quiser que este link seja renderizado, você pode definir este valor para FALSE.

Nota

Este valor também pode ser traduzido via arquivo de linguagem.

$config[‘next_tag_open’] = ‘<div>’;

A tag de abertura para o link “Próximo”.

$config[‘next_tag_close’] = ‘</div>’;

A tag de fechamento para o link “Próximo”.

Personalizando o link “Anterior”¶

$config[‘prev_link’] = ‘&lt;’;

O texto que será exibido no link da página “anterior”. Se não quiser que este link seja renderizado, você alterar este valor para FALSE.

Nota

Este valor também pode ser traduzido via arquivo de linguagem.

$config[‘prev_tag_open’] = ‘<div>’;

A tag de abertura para o link “Anterior”.

$config[‘prev_tag_close’] = ‘</div>’;

A tag de fechamento para o link “Anterior”.

Personalizando o link de “Página Atual”¶

$config[‘cur_tag_open’] = ‘<b>’;

A tag de abertura para o link de página “atual”.

$config[‘cur_tag_close’] = ‘</b>’;

A tag de fechamento para o link de página “atual”.

Personalizando o link do “Digito”¶

$config[‘num_tag_open’] = ‘<div>’;

A tag de abertura para o link “dígito”.

$config[‘num_tag_close’] = ‘</div>’;

A tag de fechamento para o link “digito”.

Ocultando as Páginas¶

Se deseja não listar páginas específicas (por exemplo, se quer somente os links de “próximo” e “anterior”), você pode suprimir a renderização das páginas adicionando:

$config['display_pages'] = FALSE;

Adicionando atributos aos links¶

Se deseja adicionar um atributo extra a cada link renderizado pela classe de paginação, você pode defini-los como pares de chave/valor na configuração dos “atributos ”:

// Resulta: class="myclass"
$config['attributes'] = array('class' => 'myclass');

Nota

O uso do método antigo de definir classes via “anchor_class” está obsoleto.

Desabilitando o atributo “rel”¶

Por padrão, o atributo rel é gerado dinamicamente e anexado aos links apropriados. Se por alguma razão você deseja desligar isso, você pode passar FALSE como atributo regular

$config['attributes']['rel'] = FALSE;
