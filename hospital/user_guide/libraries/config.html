Classe de Configuração¶

A classe Config fornece um meio para recuperar as preferências de configuração. Essas preferências podem vir do arquivo de configuração padrão (application/config/config.php) ou de seus próprios arquivos de configuração personalizados.

Nota

Esta classe é inicializada automaticamente pelo sistema, portanto, não há necessidade de fazê-lo manualmente.

Trabalhando com a Classe de Configuração¶

Anatomia do Arquivo de Configuração¶

Por padrão, o CodeIgniter possui um arquivo de configuração primário, localizado em application/config/config.php. Se você abrir o arquivo usando seu editor de texto, você verá que os itens de configuração estão armazenados em uma matriz chamado $config.

Você pode adicionar seus próprios itens de configuração a este arquivo, ou se você preferir manter seus itens de configuração separados (supondo que você precise de itens de configuração), basta criar seu próprio arquivo e salvá-lo na pasta de configuração.

Nota

Se você criar seus próprios arquivos de configuração, use o mesmo formato que o principal, armazenando seus itens em uma matriz chamado $config. CodeIgniter irá gerenciar esses arquivos com inteligência para que não haja conflito mesmo que a matriz tenha o mesmo nome (assumindo que um índice de matriz não é o mesmo que outro).

Carregando o Arquivo de Configuração¶

Nota

O CodeIgniter carrega automaticamente o arquivo de configuração primário (application/config/config.php), então você só precisará carregar um arquivo de configuração se você tiver criado o seu próprio.

Existem duas maneiras de carregar um arquivo de configuração:

Carregamento Manual¶

Para carregar um dos seus arquivos de configuração personalizados, você usará a seguinte função dentro do controlador que o precisa: 

$this->config->load('filename');

Onde o nome do arquivo é o nome do seu arquivo de configuração, sem a extensão do arquivo .php.

Se você precisa carregar vários arquivos de configuração normalmente eles serão mesclados em uma matriz de configuração principal. As colisões de nomes podem ocorrer, no entanto, se você tiver identificado nomes de matriz idênticos em diferentes arquivos de configuração. Para evitar colisões, você pode definir o segundo parâmetro como TRUE e cada arquivo de configuração será armazenado em um índice de matriz correspondente ao nome do arquivo de configuração. Exemplo:

// Armazenado em uma matriz com este protótipo: $this->config['blog_settings'] = $config
$this->config->load('blog_settings', TRUE);

Consulte a seção intitulada obtendo os itens de configuração abaixo para saber como recuperar os itens de configuração definidos dessa maneira.

O terceiro parâmetro permite que você suprima erros no caso de um arquivo de configuração não existir:

$this->config->load('blog_settings', FALSE, TRUE);

Auto-Careegamento¶

Se você achar que precisa de um arquivo de configuração específico globalmente, pode fazê-lo carregar automaticamente pelo sistema. Para fazer isso, abra o arquivo autoload.php, localizado em application/config/autoload.php, e adicione seu arquivo de configuração conforme indicado no arquivo.

Obtendo Itens de configuração¶

Para recuperar um item do seu arquivo de configuração, use a seguinte função:

$this->config->item('item_name');

Onde item_name é o índice de matriz do $config que deseja recuperar. Por exemplo, para buscar sua escolha de idioma, você fará isso:

$lang = $this->config->item('language');

A função retorna NULL se o item que você está tentando buscar não existe.

Se você estiver usando o segundo parâmetro da função $this->config->load para atribuir seus itens de configuração a um índice específico, você pode recuperá-lo especificando o nome do índice no segundo parâmetro da função $this->config->item(). Exemplo:

// Carrega um arquivo de configuração chamado blog_settings.php e atribui-lo a um índice chamado "blog_settings"
$this->config->load('blog_settings', TRUE);

// Recupere um item de configuração denominado site_name contido na matriz do blog_settings
$site_name = $this->config->item('site_name', 'blog_settings');

// Uma maneira alternativa de especificar o mesmo item:
$blog_config = $this->config->item('blog_settings');
$site_name = $blog_config['site_name'];

Setando um Item de Configuração¶

Se você quiser configurar dinamicamente um item de configuração ou alterar um existente, você pode fazê-lo usando:

$this->config->set_item('item_name', 'item_value');

Onde item_name é o índice da matriz do $config que deseja alterar e item_value é seu valor.

Ambientes¶

Você pode carregar diferentes arquivos de configuração dependendo do ambiente atual. A constante ENVIRONMENT é definida em index.php e é descrita em detalhes na seção Ambientes de Manipulação.

Para criar um arquivo de configuração específico do ambiente, crie ou copie um arquivo de configuração em application/config/{ENVIRONMENT}/{FILENAME}.php

Por exemplo, para criar um config.php somente de produção, você deve:

  1. Criar o diretório application/config/production/
  2. Copie seu config.php existente para o diretório acima
  3. Edite em application/config/production/config.php para que ele contenha suas configurações de produção

Quando você configura a constante ENVIRONMENT para 'produção', as configurações para o seu novo config.php somente de produção serão carregadas.

Você pode colocar os seguintes arquivos de configuração em pastas específicas do ambiente:

  • Arquivos de configuração padrão do CodeIgniter
  • Seus próprios arquivos de configuração personalizados

Nota

CodeIgniter sempre carrega primeiro o arquivo de configuração global (ou seja, o application/config/), então tenta carregar os arquivos de configuração para o ambiente atual. Isso significa que você não é obrigado a colocar todos os seus arquivos de configuração em uma pasta de ambiente. Somente os arquivos que mudam por ambiente. Além disso, você não precisa copiar todos os itens de configuração no arquivo de configuração do ambiente. Apenas os itens de configuração que deseja alterar para o seu ambiente. Os itens de configuração declarados em suas pastas de ambiente sempre substituem aqueles em seus arquivos de configuração globais.

Referência de Classe¶

class CI_Config¶
$config

Matriz de todos os valores de configuração carregados

$is_loaded

Matriz de todos os arquivos de configuração carregados

item($item[, $index=''])¶
Parâmetros:
  • $item (string) – Nome do item de configuração
  • $index (string) – Nome do índice
Retorna:

Valor do item de configuração ou NULL se não for encontrado
Tipo de retorno:

mixed

Obtenha um item de arquivo de configuração.

set_item($item, $value)¶
Parâmetros:
  • $item (string) – Nome do item de configuração
  • $value (string) – Valor do item de configuração
Tipo de retorno:

void

Define um item de arquivo de configuração para o valor especificado.

slash_item($item)¶
Parâmetros:
  • $item (string) – Nome do item de configuração
Retorna:

Configurar o valor do item com uma barra a seguir ou NULL se não for encontrado
Tipo de retorno:

mixed

Este método é idêntico ao item(), exceto que ele anexa uma barra inclinada para o final do item, se existir.

load([$file = ''[, $use_sections = FALSE[, $fail_gracefully = FALSE]]])¶
Parâmetros:
  • $file (string) – Nome do arquivo de configuração
  • $use_sections (bool) – Se os valores de configuração devem ser carregados em sua própria seção (índice da matriz de configuração principal)
  • $fail_gracefully (bool) – Se deseja retornar FALSE ou para exibir uma mensagem de erro
Retorna:

TRUE em caso de sucesso, FALSE em falha
Tipo de retorno:

booleano

Carrega um arquivo de configuração.

site_url()¶
Retorna:Site URL
Tipo de retorno:string

Este método recupera o URL para o seu site, juntamente com o valor "índice" que você especificou no arquivo de configuração.

Esse método é normalmente acessado através das funções correspondentes em Auxiliar de URL.

base_url()¶
Retorna:Base URL
Tipo de retorno:string

Este método recupera o URL para o seu site, além de um caminho opcional, como uma folha de estilo ou imagem.

Esse método é normalmente acessado através das funções correspondentes em Auxiliar de URL.

system_url()¶
Retorna:A URL apontando para o seu diretório system/ do CI
Tipo de retorno:string

Este método recupera a URL do seu sistema/diretório CodeIgniter.

Nota

Este método é DEPRECIADO porque incentiva o uso de práticas de codificação inseguras. O seu system/directory não deve ser acessível publicamente.
