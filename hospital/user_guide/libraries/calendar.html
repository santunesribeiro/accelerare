Classe de Calendário¶

A classe Calendário permite que você crie calendários dinamicamente. Os seus calendários podem ser formatados através do uso de um modelo de calendário, permitindo o controle de 100% em todos os aspectos do seu design. Além disso, você pode passar dados para suas células do calendário.

Usando a Classe de Calendário¶

Inicializando a Classe¶

Como a maioria das outras classes em CodeIgniter, a classe Calendar é inicializada no seu controlador usando a função $this->load->library:

$this->load->library('calendar');

Uma vez carregado, o objeto Calendário estará disponível usando:

$this->calendar

Mostrando o Calendário¶

Aqui está um exemplo muito simples que mostra como você pode exibir um calendário:

$this->load->library('calendar');
echo $this->calendar->generate();

O código acima gerará um calendário para o mês/ano atual com base no tempo do servidor. Para mostrar um calendário por um mês e ano específicos, você passará essas informações para a função de geração do calendário:

$this->load->library('calendar');
echo $this->calendar->generate(2006, 6);

O código acima irá gerar um calendário que mostra o mês de junho de 2006. O primeiro parâmetro especifica o ano, o segundo parâmetro especifica o mês.

Passando Dados para as Células do Calendário¶

Para adicionar dados às células do calendário, é necessário criar uma matriz associativa em que as chaves correspondam aos dias que você deseja preencher e o valor da matriz contém os dados. A matriz é passada para o terceiro parâmetro da função de geração de calendário. Considere este exemplo:

$this->load->library('calendar');

$data = array(
        3  => 'http://example.com/news/article/2006/06/03/',
        7  => 'http://example.com/news/article/2006/06/07/',
        13 => 'http://example.com/news/article/2006/06/13/',
        26 => 'http://example.com/news/article/2006/06/26/'
);

echo $this->calendar->generate(2006, 6, $data);

Usando o exemplo acima, os números de dias 3, 7, 13 e 26 se tornarão links apontando para os URLs que você forneceu.

Nota

Por padrão, assume-se que sua matriz conterá links. Na seção que explica o modelo de calendário abaixo, você verá como você pode personalizar como os dados passados ​​para suas células são tratados para que você possa passar diferentes tipos de informações.

Configurando Preferências de Exibição¶

Existem sete preferências que você pode configurar para controlar vários aspectos do calendário. As preferências são definidas passando uma série de preferências no segundo parâmetro da função de carregamento. Aqui está um exemplo:

$prefs = array(
        'start_day'    => 'saturday',
        'month_type'   => 'long',
        'day_type'     => 'short'
);

$this->load->library('calendar', $prefs);

echo $this->calendar->generate();

O código acima começaria o calendário no sábado, usaria o título "longo" do mês e os nomes dos dias "curtos". Mais informações sobre as preferências abaixo.

Preferência Padrão Opções Descrição
template None None
Uma string ou um array contendo o modelo de calendário.
Veja a seção de modelo abaixo.
local_time time() None Um timestamp Unix correspondente à hora atual.
start_day sunday Qualquer dia de semana (sunday, monday, tuesday, etc.) Define o dia da semana em que o calendário deve começar.
month_type long long, short
Determina a versão do nome do mês a ser usada no cabeçalho.
long = Janeiro, short = Jan.
day_type abr long, short, abr
Determina a versão dos nomes dos dias da semana a serem usados
​​nos cabeçalhos das colunas. long = Segunda, short = Seg, abr = Su.
show_next_prev FALSE TRUE/FALSE (boolean)
Determina se deseja exibir links que permitem alternar para os meses
seguintes/anteriores. Veja informações sobre este recurso abaixo.
next_prev_url controller/method A URL Define o caminho de base utilizado nos links do calendário seguinte/anterior.
show_other_days FALSE TRUE/FALSE (boolean)
Determina se deve exibir dias de outros meses que compartilhem a
Primeira ou última semana do mês do calendário.

Mostrando Links do Mês Seguinte/Anterior¶

Para permitir que seu calendário incremente ou diminua dinamicamente através dos links seguintes/anteriores, você deve configurar seu código de calendário semelhante a este exemplo:

$prefs = array(
        'show_next_prev'  => TRUE,
        'next_prev_url'   => 'http://example.com/index.php/calendar/show/'
);

$this->load->library('calendar', $prefs);

echo $this->calendar->generate($this->uri->segment(3), $this->uri->segment(4));

Você notará algumas coisas sobre o exemplo acima:

  • Você deve configurar o "show_next_prev" para TRUE.
  • Você deve fornecer o URL ao controlador que contém seu calendário na preferência "próximo_prev_url". Se você não estiver, ele será definido como o atual controlador/método.
  • Você deve fornecer o "ano" e o "mês" para a função de geração de calendário através dos segmentos URI onde eles aparecem (Nota: A classe do calendário adiciona automaticamente o ano/mês ao URL básico que você fornece).

Criando um Modelo de Calendário¶

Ao criar um modelo de calendário, você possui 100% de controle sobre o design do seu calendário. Usando o método de seqüência, cada componente do seu calendário será colocado dentro de um par de pseudo-variáveis ​​como mostrado aqui:

$prefs['template'] = '

        {table_open}<table border="0" cellpadding="0" cellspacing="0">{/table_open}

        {heading_row_start}<tr>{/heading_row_start}

        {heading_previous_cell}<th><a href="{previous_url}">&lt;&lt;</a></th>{/heading_previous_cell}
        {heading_title_cell}<th colspan="{colspan}">{heading}</th>{/heading_title_cell}
        {heading_next_cell}<th><a href="{next_url}">&gt;&gt;</a></th>{/heading_next_cell}

        {heading_row_end}</tr>{/heading_row_end}

        {week_row_start}<tr>{/week_row_start}
        {week_day_cell}<td>{week_day}</td>{/week_day_cell}
        {week_row_end}</tr>{/week_row_end}

        {cal_row_start}<tr>{/cal_row_start}
        {cal_cell_start}<td>{/cal_cell_start}
        {cal_cell_start_today}<td>{/cal_cell_start_today}
        {cal_cell_start_other}<td class="other-month">{/cal_cell_start_other}

        {cal_cell_content}<a href="{content}">{day}</a>{/cal_cell_content}
        {cal_cell_content_today}<div class="highlight"><a href="{content}">{day}</a></div>{/cal_cell_content_today}

        {cal_cell_no_content}{day}{/cal_cell_no_content}
        {cal_cell_no_content_today}<div class="highlight">{day}</div>{/cal_cell_no_content_today}

        {cal_cell_blank}&nbsp;{/cal_cell_blank}

        {cal_cell_other}{day}{/cal_cel_other}

        {cal_cell_end}</td>{/cal_cell_end}
        {cal_cell_end_today}</td>{/cal_cell_end_today}
        {cal_cell_end_other}</td>{/cal_cell_end_other}
        {cal_row_end}</tr>{/cal_row_end}

        {table_close}</table>{/table_close}
';

$this->load->library('calendar', $prefs);

echo $this->calendar->generate();

Usando o método array, você passará um par de chave => valor. Você pode passar tantos ou poucos valores como você gostaria. As chaves omitidas usarão os valores padrão herdados na classe do calendário.

Exemplo:

$prefs['template'] = array(
        'table_open'           => '<table class="calendar">',
        'cal_cell_start'       => '<td class="day">',
        'cal_cell_start_today' => '<td class="today">'
);

$this->load->library('calendar', $prefs);

echo $this->calendar->generate();

Referência da Classe¶

class CI_Calendar¶
initialize([$config = array()])¶
Parâmetros:
  • $config (array) – Parâmetros de configuração
Retorna:

CI_Calendar Instância (encadeamento do método)
Tipo de retorno:

CI_Calendar

Inicializa as preferências do Calendário. Aceita uma matriz associativa como entrada, contendo preferências de exibição.

generate([$year = ''[, $month = ''[, $data = array()]]])¶
Parâmetros:
  • $year (int) – Ano
  • $month (int) – Mês
  • $data (array) – Dados a serem mostrados nas células do calendário
Retorna:

Calendário formatado em HTML
Tipo de retorno:

string

Gere o calendário.

get_month_name($month)¶
Parâmetros:
  • $month (int) – Mês
Retorna:

Nome do mês
Tipo de retorno:

string

Gera um nome de mês textual com base no mês numérico fornecido.

get_day_names($day_type = '')¶
Parâmetros:
  • $day_type (string) – ‘long’, ‘short’, ou ‘abr’
Retorna:

Array dos nomes dos dias
Tipo de retorno:

array

Retorna uma série de nomes de dias (domingo, segunda-feira, etc.) com base no tipo fornecido. Opções: longa, curta. Se não for fornecido $day_type (ou se for fornecido um tipo inválido), esse método retornará o estilo "abreviado".

adjust_date($month, $year)¶
Parâmetros:
  • $month (int) – Mês
  • $year (int) – Ano
Retorna:

Um array associativo contendo mês e ano
Tipo de retorno:

array

Este método garante que você tenha um mês/ano válido. Por exemplo, se você enviar 13 como o mês, o ano aumentará e o mês se tornará janeiro:

print_r($this->calendar->adjust_date(13, 2014));

saída:

Array
(
        [month] => '01'
        [year] => '2015'
)
get_total_days($month, $year)¶
Parâmetros:
  • $month (int) – Mês
  • $year (int) – Ano
Retorna:

Contagem de dias no mês especificado
Tipo de retorno:

int

Total de dias em um determinado mês:

echo $this->calendar->get_total_days(2, 2012);
// 29

Nota

Este método é um alias para a função days_in_month() do Auxiliar de Data.

default_template()¶
Retorna:Um array do modelo de valor
Tipo de retorno:array

Define o modelo padrão. Esse método é usado quando você não criou seu próprio modelo.

parse_template()¶
Retorna:CI_Calendar Instância (encadeamento do método)
Tipo de retorno:CI_Calendar

Colhe os dados dentro do modelo {pseudo-variables} usados para exibir o calendário.
