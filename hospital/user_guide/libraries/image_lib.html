Classe de Manipulação de Imagem¶

A classe de Manipulação de Imagens do CodeIgniter permite executar as seguintes ações com imagens:

  • Redimensionamento
  • Criação de miniaturas (thumbnails)
  • Corte (cropping)
  • Rotação
  • Inserção de marca d'água (watermarking)

As três principais bibliotecas de imagens são suportadas: GD/GD2, NetPBM e ImageMagick.

Nota

A criação de marcas d'água apenas está disponível quando usada a biblioteca GD/GD2. Mesmo que outras bibliotecas sejam suportadas, a GD é necessária para o cálculo de propriedades da imagem. O processamento da imagem, entretanto, poderá ser realizado em qualquer outra biblioteca suportada.

Inicializando a Classe¶

Assim como a maioria das outras classes no CodeIgniter, a classe Image é inicializada em seu controller usando o método $this->load->library:

$this->load->library('image_lib');

Uma vez que a biblioteca esteja carregada, ela estará pronta para o uso. O objeto da biblioteca de imagens que você utilizará para chamar todas as funções é: $this->image_lib

Processando uma imagem¶

Independentemente do tipo de processamento que você pretenda realizar (redimensionar, cortar, girar ou inserir marca d’água), o processo geral é o mesmo. Você definirá algumas preferências de acordo com a ação que você pretende executar e, então, chamará um dos quatro métodos de processamento disponíveis. Por exemplo, para criar uma miniatura de uma imagem, você deverá fazer assim:

$config['image_library'] = 'gd2';
$config['source_image'] = '/path/to/image/mypic.jpg';
$config['create_thumb'] = TRUE;
$config['maintain_ratio'] = TRUE;
$config['width']         = 75;
$config['height']       = 50;

$this->load->library('image_lib', $config);

$this->image_lib->resize();

O código acima informa ao método de redimensionamento de imagem para que procure por uma imagem chamada mypic.jpg, localizada no diretório source_image, e crie uma miniatura de 75 x 50 pixels usando a biblioteca GD2. A opção maintain_ratio habilitada garante que a miniatura será criada com a maior proximidade possível às dimensões definidas, no entanto, deverá preservar as proporções da imagem original. A miniatura será chamada mypic_thumb.jpg e será armazenada no mesmo diretório que source_image.

Nota

Para que a classe de manipulação de imagens realize o processamento, o diretório onde serão armazenadas as imagens deve possuir permissão de escrita.

Nota

O processamento de imagens pode demandar uma quantidade considerável de memória do servidor para algumas operações. Caso se depare com erros de estouro de memória enquanto realiza o processamento, talvez seja necessário limitar o tamanho máximo das imagens e/ou ajustar os limites de memória do PHP.

Métodos de Processamento¶

Existem quatro métodos de processamento disponíveis:

  • $this->image_lib->resize()
  • $this->image_lib->crop()
  • $this->image_lib->rotate()
  • $this->image_lib->watermark()

Os métodos retornam o valor booleano TRUE em caso de sucesso e FALSE em caso de falha. Em caso de falha, você pode obter a mensagem de erro usando esta função:

echo $this->image_lib->display_errors();

Uma boa prática é usar o método de processamento dentro de uma estrutura condicional, mostrando um erro em caso de falha:

if ( ! $this->image_lib->resize())
{
        echo $this->image_lib->display_errors();
}

Nota

Opcionalmente, você pode especificar a formatação HTML para ser aplicada aos erros. Para isso, forneça as tags de abertura e fechamento como parâmetros:

$this->image_lib->display_errors('<p>', '</p>');

Preferências¶

As preferências abaixo descritas permitem adequar o processamento de imagens às suas necessidades.

Perceba que nem todas as preferências estão disponíveis para cada função. Por exemplo, as preferências de eixo x/y estão disponíveis apenas para o recurso de corte da imagem. Da mesma forma, as preferências de largura e altura não tem qualquer efeito na função de corte. Na tabela a seguir, a coluna “disponibilidade” indica quais funções suportam cada definição de preferência.

Legenda da disponibilidade:

  • R – Redimensionamento
  • C – Corte
  • X – Rotação
  • W – Marca d’água
Preferência Valor Padrão Opções Descrição Disponibilidade
image_library GD2 GD, GD2, ImageMagick, NetPBM Define a biblioteca de manipulação de imagens a ser utilizada. R, C, X, W
library_path Nenhum Nenhum Define o caminho para as bibliotecas ImageMagick ou NetPBM. Caso utilize uma destas bibliotecas, você necessitará fornecer o caminho da instalação. R, C, X R, C, S, W
source_image Nenhum Nenhum Define o nome e caminho para onde o arquivo da imagem de origem está armazenada. O caminho deve ser relativo ou absoluto e não uma URL.  
dynamic_output FALSE TRUE/FALSE (boolean) Determina se o novo arquivo de imagem deve ser escrito no disco ou gerado dinamicamente. Nota: caso escolha a opção “dinâmica”, apenas uma imagem poderá ser gerada por vez e esta imagem não poderá ser posicionada na página. A imagem de saída será direcionada automaticamente ao navegador, juntamente com os cabeçalhos apropriados. R, C, X, W
file_permissions 0644 (integer) Nível de permissão a ser aplicada aos arquivos escritos em disco gerados após o processamento da imagem. ATENÇÃO: Utilize a notação octal! R, C, X, W
quality 90% 1 - 100% Define a qualidade da imagem. Quanto maior a qualidade, maior também será o tamanho do arquivo. R, C, X, W
new_image Nenhum Nenhum Define o nome e caminho onde será escrita a imagem resultado do processamento. Você utilizará esta preferência quando criar uma imagem cópia. O caminho deve ser absoluto ou relativo e não uma URL. R, C, X, W
width Nenhum Nenhum Define a largura esperada para a imagem de saída. R, C
height Nenhum Nenhum Define a altura esperada para a imagem de saída. R, C
create_thumb FALSE TRUE/FALSE (boolean) Informa ao método de processamento de imagem para que crie uma miniatura. R
thumb_marker _thumb Nenhum Especifica a sequência de caracteres indicadora da miniatura. Ela será inserida logo após à extensão do arquivo. Logo, mypic.jpg se tornará mypic_thumb.jpg R
maintain_ratio TRUE TRUE/FALSE (boolean) Especifica se deverá ser mantida a proporção original da imagem quando redimensionada ou se os valores de altura e largura deverão ser seguidos com rigor. R, C
master_dim auto auto, width, height Especifica qual será o eixo principal quando redimensionando ou criando miniaturas para a imagem. Por exemplo, digamos que você redimensione uma imagem para 100 x 75 pixels. Se as dimensões da imagem original não permitirem um perfeito redimensionamento a estas dimensões, esta preferência determina qual eixo deverá ser usado como o valor absoluto. A opção “auto” determina o eixo automaticamente, baseado em qual dimensão é maior (largura ou altura). R
rotation_angle Nenhum 90, 180, 270, vrt, hor Determina o ângulo de rotação. Note que o PHP realiza a rotação no sentido anti-horário, então uma rotação de 90 graus para a direita deve ser especificada como 270 graus. X
x_axis Nenhum Nenhum Define a coordenada X, em pixels, para o corte da imagem. Por exemplo, uma definição com o valor “30” irá cortar a imagem com uma borda interna de 30 pixels à esquerda. C
y_axis Nenhum Nenhum Define a coordenada Y, em pixels, para o corte da imagem. Por exemplo, uma definição com o valor “30” irá cortar a imagem com uma borda interna de 30 pixels a partir do topo. C

Definindo preferências em um arquivo de configuração¶

Caso prefira não definir preferências utilizando o método acima, você poderá colocá-las em um arquivo de configuração. Simplesmente crie um novo arquivo chamado image_lib.php e adicione o array $config neste arquivo. Armazene o arquivo em /config/image_lib.php e ele será reconhecido automaticamente. Você não necessitará utilizar o método $this->image_lib->initialize() caso guarde as preferências em um arquivo de configuração.

Inserção de Marcas d’água¶

O recurso de marcas d’água requer as bibliotecas GD/GD2.

Dois tipos de marcas d’água¶

Existem dois tipos de marcas d’água que você poderá utilizar:

  • Texto: a marca d’água será gerada usando texto, com uma fonte True Type definida por você ou utilizando texto nativo de saída suportado pela biblioteca GD. Caso utilize a versão True Type, sua instalação da biblioteca GD deverá ser compilada com a opção de suporte a True Type (geralmente esta opção é habilitada por padrão, porém não sempre).
  • Sobreposição: a marca d’água será gerada pela sobreposição de uma imagem em relação à outra. Geralmente, um PNG ou GIF transparente contém a marca d’água que será impressa sobre a imagem a ser processada.

Inserindo uma marca d’água¶

Assim como os outros métodos (redimensionamento, corte e rotação), o processo geral para a inserção de marcas d’água envolve a definição de preferências de acordo com a ação que você pretende realizar e, posteriormente, a chamada ao método de criação de marca d’água. Aqui está um exemplo:

$config['source_image'] = '/path/to/image/mypic.jpg';
$config['wm_text'] = 'Copyright 2006 - John Doe';
$config['wm_type'] = 'text';
$config['wm_font_path'] = './system/fonts/texb.ttf';
$config['wm_font_size'] = '16';
$config['wm_font_color'] = 'ffffff';
$config['wm_vrt_alignment'] = 'bottom';
$config['wm_hor_alignment'] = 'center';
$config['wm_padding'] = '20';

$this->image_lib->initialize($config);

$this->image_lib->watermark();

O exemplo acima utilizará uma fonte True Type de 16 pixels para criar o texto “Copyright 2006 - John Doe”. A marca d’água será posicionada à base e ao centro da imagem, 20 pixels acima da borda inferior.

Nota

Para que a classe de imagem esteja apta a realizar o processamento, o arquivo da imagem deverá possuir permissão de escrita. Por exemplo, 777.

Preferências de Marca d’Água¶

Esta tabela exibe as preferências que estão disponíveis para ambos os tipos de marcas d’água (texto e sobreposição):

Preferência Valor Padrão Opções Descrição
wm_type text text, overlay Define o tipo de marca d’água que deverá ser utilizado.
source_image Nenhum Nenhum Define o caminho e nome para o arquivo de origem. O caminho deve ser relativo ou absoluto, não uma URL.
dynamic_output FALSE TRUE/FALSE (boolean) Determina se o novo arquivo de imagem deve ser escrito no disco ou gerado dinamicamente. Nota: caso escolha a opção “dinâmica”, apenas uma imagem poderá ser gerada por vez e esta imagem não poderá ser posicionada na página. A imagem de saída será direcionada automaticamente ao navegador, juntamente com os cabeçalhos apropriados.
quality 90% 1 - 100% Define a qualidade da imagem. Quanto maior a qualidade, maior o tamanho do arquivo.
wm_padding Nenhum Um número A quantidade de preenchimento, definida em pixels, que será aplicada à marca d’água para afastá-la das bordas da imagem.
wm_vrt_alignment bottom top, middle, bottom Define o alinhamento vertical para a imagem da marca d’água.
wm_hor_alignment center left, center, right Define o alinhamento horizontal para a imagem da marca d’água.
wm_hor_offset Nenhum Nenhum Você pode especificar um deslocamento horizontal, em pixels, a ser aplicado à posição da marca d’água. Normalmente, o deslocamento move a marca d’água para a direita, exceto caso você tenha o seu alinhamento definido para “right”, o que fará com que o valor do deslocamento desloque a marca d´água para a esquerda.
wm_vrt_offset Nenhum Nenhum Você pode especificar um deslocamento vertical, em pixels, a ser aplicado à posição da marca d’água. Normalmente, o deslocamento move a marca d’água para baixo, exceto caso você tenha o seu alinhamento definido para “bottom”, o que fará com que o valor do deslocamento desloque a marca d´água para cima.

Preferências de texto¶

Esta tabela mostra as preferências que estão disponíveis para as marcas d’água do tipo texto:

Preferência Valor Padrão Opções Descrição
wm_text Nenhum Nenhum O texto que você gostaria de exibir como marca d’água. Tipicamente, uma informação sobre direitos autorais.
wm_font_path Nenhum Nenhum O caminho no servidor para a fonte True Type que você gostaria de utilizar. Caso você não defina esta preferência, a fonte nativa da biblioteca GD será utilizada.
wm_font_size 16 Nenhum O tamanho do texto. Nota: caso você não esteja usando a opção True Type acima, o número é definido usando um intervalo entre 1 e 5. Caso contrário, você pode usar qualquer tamanho de pixels válido para a fonte que você está utilizando.
wm_font_color ffffff Nenhum A cor da fonte, especificada em hexadecimais. É possível informar esta opção com seis caracteres (por exemplo, 993300) ou, abreviadamente, com três caracteres (por exemplo, fff).
wm_shadow_color Nenhum Nenhum A cor da sombra, especificada em hexadecimais. Caso deixe esta opção em branco, nenhuma sombra será utilizada. É possível informar esta opção com seis caracteres (por exemplo, 993300) ou, abreviadamente, com três caracteres (por exemplo, fff).
wm_shadow_distance 3 Nenhum A distância, em pixels, entre a fonte e a sombra.

Preferências de Sobreposição¶

Esta tabela mostra as preferências que estão disponíveis para a marca d’água do tipo sobreposição:

Preferência Valor Padrão Opções Descrição
wm_overlay_path Nenhum Nenhum O caminho no servidor para a imagem que você deseja utilizar como marca d’água. Obrigatório apenas caso esteja usando o método de sobreposição.
wm_opacity 50 1 - 100 A opacidade da imagem. Você pode especificar a transparência da imagem da marca d’água. Esta opção permite que a marca d’água não seja tão escura a ponto de impedir a visualização de detalhes da imagem original que está por baixo dela. Um valor comum para a opacidade é 50%.
wm_x_transp 4 Um número Caso você esteja inserindo marca d’água em uma imagem PNG ou GIF, você deve especificar uma cor na imagem para ser “transparente”. Esta opção, em conjunto com a que está a seguir, permitirá que você especifique esta cor. Funciona assim: você especifica as coordenadas “X” e “Y”, medidas a partir do canto superior esquerdo da imagem, do pixel que contém a cor que você pretende que seja transparente.
wm_y_transp 4 Um número Em conjunto com a configuração anterior, esta opção permite definir a coordenada do pixel que representa a cor que você pretende que seja transparente.

Referência de Classe¶

classe CI_Image_lib¶
initialize([$props = array()])¶
Parâmetros:
  • $props (array) – Preferências do processamento de imagem
Retorna:

TRUE em caso de sucesso, FALSE em caso de configurações inválidas.
Tipo de retorno:

bool

Inicializa a classe para o processamento de uma imagem.

resize()¶
Retorna:TRUE em sucesso, FALSE em falha.
Tipo de retorno:bool

O método de redimensionamento de imagem permite que você redimensione a imagem original, crie uma cópia (com ou sem redimensionamento) ou uma miniatura da imagem.

Para propósitos práticos, não há diferença entre criar uma cópia e criar uma miniatura, exceto que a miniatura terá uma string indicadora concatenada como parte de seu nome (por exemplo, mypic_thumb.jpg).

Todas as preferências listadas na tabela de preferências estão disponíveis para este método, exceto três: rotation_angle, x_axis e y_axis.

Criando uma miniatura

O método de redimensionamento criará um arquivo de miniatura (e preservará o original) caso você defina esta preferência para TRUE:

$config['create_thumb'] = TRUE;

Esta preferência determina se uma miniatura é criada ou não.

Criando uma cópia

O método de redimensionamento irá criar uma cópia do arquivo da imagem (e preservar o original) caso você defina um caminho e/ou um novo nome para o arquivo usando esta preferência:

$config['new_image'] = '/path/to/new_image.jpg';

Notas sobre essa preferência:

  • Caso somente o nome da nova imagem seja especificado, a imagem gerada será armazenada no mesmo diretório da original;
  • Caso somente o caminho seja especificado, a nova imagem será armazenada no destino com o mesmo nome da original;
  • Caso sejam especificados o caminho e o nome, a nova imagem será armazenada no caminho especificado e receberá o nome, conforme escolhido.

Redimensionando a imagem original

Caso nenhuma das opções acima descritas (create_thumb e new_image) seja utilizada, o método de redimensionamento irá processar o arquivo da imagem original e substituí-lo pela nova imagem.

crop()¶
Retorna:TRUE em sucesso, FALSE em falha
Tipo de retorno:bool

O método de corte funciona de maneira muito semelhante ao método de redimensionamento, exceto pelo fato de exigir que você defina as preferências para os eixos X e Y, em pixels, especificando o ponto de corte:

$config['x_axis'] = 100;
$config['y_axis'] = 40;

Todas as preferências listadas na tabela de Preferências estão disponíveis para este método, exceto rotation_angle, create_thumb e new_image.

Aqui está um exemplo mostrando como cortar uma imagem:

$config['image_library'] = 'imagemagick';
$config['library_path'] = '/usr/X11R6/bin/';
$config['source_image'] = '/path/to/image/mypic.jpg';
$config['x_axis'] = 100;
$config['y_axis'] = 60;

$this->image_lib->initialize($config);

if ( ! $this->image_lib->crop())
{
        echo $this->image_lib->display_errors();
}

Nota

Sem uma interface visual, é muito difícil cortar uma imagem. A utilidade desta função é nula, exceto caso você tenha em mente construir tal interface. Foi exatamente isso que nós fizemos no módulo de galeria de fotos do ExpressionEngine, o sistema gerenciador de conteúdo que nós desenvolvemos. Nós adicionamos uma interface gráfica para o usuário, construída em Javascript, que permite a seleção da área a ser cortada.

rotate()¶
Retorna:TRUE em sucesso, FALSE em falha.
Tipo de retorno:bool

O método de rotação de imagem exige que o ângulo da rotação seja definido nas preferências:

$config['rotation_angle'] = '90';

Existem cinco opções de rotação:

  1. 90 – Rotação em 90 graus anti-horário;
  2. 180 – Rotação em 180 graus anti-horário;
  3. 270 – Rotação em 270 graus anti-horário;
  4. hor – Inverte a imagem horizontalmente;
  5. vrt – Inverte a imagem verticalmente.

Aqui está um exemplo mostrando como você pode fazer a rotação de uma imagem:

$config['image_library'] = 'netpbm';
$config['library_path'] = '/usr/bin/';
$config['source_image'] = '/path/to/image/mypic.jpg';
$config['rotation_angle'] = 'hor';

$this->image_lib->initialize($config);

if ( ! $this->image_lib->rotate())
{
        echo $this->image_lib->display_errors();
}

clear()¶
Tipo de retorno:void

O método “clear” redefine todos os valores usados no processamento de imagens. Você deverá invocá-lo caso esteja processando imagens dentro de um loop.

$this->image_lib->clear();
