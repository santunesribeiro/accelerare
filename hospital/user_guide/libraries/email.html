Classe de Email¶

A Classe de Email robusta do CodeIgniter suporta os seguintes recursos:

  • Múltiplos protocolos: Mail, Sendmail e SMTP
  • Criptografia TLS e SSL para SMTP
  • Múltiplos Destinatários
  • CC e BCCs
  • HTML or Texto Plano
  • Anexos
  • Quebra de Linha
  • Prioridades
  • Modo de Lote BCC, permitindo que grandes listas de e-mail sejam divididas em pequenos lotes BCC.

Usando a Biblioteca de Email¶

Enviando Email¶

O envio de e-mail não é simples, mas você pode configurá-lo instantaneamente ou definir suas preferências em um arquivo de configuração.

Aqui está um exemplo básico que demonstra como você pode enviar e-mail. Nota: Este exemplo assume que você está enviando o e-mail de um de seus controladores.

$this->load->library('email');

$this->email->from('your@example.com', 'Your Name');
$this->email->to('someone@example.com');
$this->email->cc('another@another-example.com');
$this->email->bcc('them@their-example.com');

$this->email->subject('Email Test');
$this->email->message('Testing the email class.');

$this->email->send();

Configurando Preferências de Email¶

Existem 21 preferências diferentes disponíveis para personalizar o modo como suas mensagens de e-mail são enviadas. Você pode configurá-los manualmente como descrito aqui ou automaticamente através de preferências armazenadas em seu arquivo de configuração, descrito abaixo:

As preferências são definidas passando uma matriz de valores de preferência para o método de inicialização de e-mail. Aqui está um exemplo de como você pode definir algumas preferências:

$config['protocol'] = 'sendmail';
$config['mailpath'] = '/usr/sbin/sendmail';
$config['charset'] = 'iso-8859-1';
$config['wordwrap'] = TRUE;

$this->email->initialize($config);

Nota

A maioria das preferências tem valores padrão que serão usados ​​se você não configurá-los.

Configurando Preferências de Email em um Arquivo de Configuração¶

Se você preferir não definir preferências usando o método acima, pode colocá-las em um arquivo de configuração. Simplesmente crie um novo arquivo chamado email.php, adicione a matriz $config nesse arquivo. Em seguida, guarde o arquivo em config/email.php e será usado automaticamente. Você NÃO precisará usar o método $this->email->initialize() se você salvar suas preferências em um arquivo de configuração.

Preferência de Email¶

A lista seguinte é uma lista de todas as preferências que podem ser definidas ao enviar e-mails.

mailpath /usr/sbin/sendmail None O caminho do servidor para Sendmail.
smtp_host No Default None Endereço do servidor SMTP.
smtp_user No Default None Nome de usuário SMTP.
smtp_pass No Default None Senha SMTP.
smtp_port 25 None Porta SMTP.
smtp_timeout 5 None Tempo limite de SMTP (em segundos)..
smtp_keepalive FALSE TRUE or FALSE (booleano) Habilite conexões SMTP persistentes.
smtp_crypto No Default tls or ssl Criptografia SMTP
wordwrap TRUE TRUE or FALSE (booleano) Habilitar quebra de linha.

Sobrescrevendo Quebra de Linha¶

Se você tiver o pacote de palavras ativado (recomendado para seguir o RFC 822) e você tem um link muito longo em seu e-mail, ele também pode ser encapsulado, fazendo com que ele não seja clicável pela pessoa que o recebe. CodeIgniter permite que você substitua manualmente o encapsulamento de palavras em parte da sua mensagem como esta:

O texto de seu email que
quebra linha normalmente.

{unwrap}http://example.com/a_long_link_that_should_not_be_wrapped.html{/unwrap}

Mais um  texto que irá
quebrar normalmente.

Coloque o item que você não deseja quebrar entre as palavras: {unwrap} {/unwrap}

Referência de Classe¶

class CI_Email¶
from($from[, $name = ''[, $return_path = NULL]])¶
Parâmetros:
  • $from (string) – “De” endereço de email
  • $name (string) – “From” nome mostrado
  • $return_path (string) – Endereço de email opcional para redirecionar o e-mail não entregue para
Retorna:

Instância do CI_Email (método de encadeamento)
Tipo de retorno:

CI_Email

Define o endereço de e-mail e o nome da pessoa que envia o e-mail:

$this->email->from('you@example.com', 'Your Name');

Você também pode definir um Caminho de Retorno, para ajudar a redirecionar o correio não entregue:

$this->email->from('you@example.com', 'Your Name', 'returned_emails@example.com');

Nota

O caminho de retorno não pode ser usado se você configurou 'smtp' como seu protocolo.

reply_to($replyto[, $name = ''])¶
Parâmetros:
  • $replyto (string) – Endereço de e-mail para respostas
  • $name (string) – Exibir nome para a resposta ao endereço de e-mail
Retorna:

Instância do CI_Email (método de encadeamento)
Tipo de retorno:

CI_Email

Define o endereço de resposta. Se a informação não for fornecida, a informação no método: from é usada. Exemplo:

$this->email->reply_to('you@example.com', 'Your Name');
to($to)¶
Parâmetros:
  • $to (mixed) – String delimitada por vírgulas ou uma variedade de endereços de e-mail
Retorna:

Instância do CI_Email (método de encadeamento)
Tipo de retorno:

CI_Email

Define o (s) endereço (s) de email CC. Assim como o "para", pode ser um único e-mail, uma lista delimitada por vírgulas ou uma matriz.

$this->email->to('someone@example.com');

$this->email->to('one@example.com, two@example.com, three@example.com');

$this->email->to(
        array('one@example.com', 'two@example.com', 'three@example.com')
);
cc($cc)¶
Parâmetros:
  • $cc (mixed) – String delimitada por vírgulas ou uma variedade de endereços de e-mail
Retorna:

Instância do CI_Email (método de encadeamento)
Tipo de retorno:

CI_Email

Define o (s) endereço (s) de email CC. Assim como o "para", pode ser um único e-mail, uma lista delimitada por vírgulas ou uma matriz..

bcc($bcc[, $limit = ''])¶
Parâmetros:
  • $bcc (mixed) – String delimitada por vírgulas ou uma variedade de endereços de e-mail
  • $limit (int) – Número máximo de e-mails para enviar por lote
Retorna:

Instância do CI_Email (método de encadeamento)
Tipo de retorno:

CI_Email

Define o (s) endereço (s) de e-mail do BCC. Assim como o método to(), pode ser um único e-mail, uma lista delimitada por vírgulas ou uma matriz.

Se $limit Está definido, o "modo em lote" será ativado, que enviará os e-mails para lotes, com cada lote que não exceda o $limit especificado.

Instância do CI_Email (método de encadeamento)
Tipo de retorno:

CI_Email

Define o assunto do e-mail:

$this->email->subject('This is my subject');

Instância do CI_Email (método de encadeamento)

CI_Email

$this->email->message('This is my message');
set_alt_message($str)¶
Parâmetros:
  • $str (string) – Corpo de mensagem de e-mail alternativo
Retorna:

Instância do CI_Email (método de encadeamento)
Tipo de retorno:

CI_Email

Define o corpo da mensagem de e-mail alternativa:

$this->email->set_alt_message('This is the alternative message');

Esta é uma string de mensagem opcional que pode ser usada se você enviar e-mails com formato HTML. Ele permite que você especifique uma mensagem alternativa sem formatação HTML, que é adicionada à seqüência de cabeçalho para pessoas que não aceitam o email HTML. Se você não definir sua própria mensagem, CodeIgniter extrairá a mensagem do seu e-mail HTML e tira as tags.

clear([$clear_attachments = FALSE])¶
Parâmetros:
  • $clear_attachments (bool) – Se deseja ou não limpar os anexos
Retorna:

Instância do CI_Email (método de encadeamento)
Tipo de retorno:

CI_Email

Inicializa todas as variáveis ​​de email em um estado vazio. Este método destina-se a ser utilizado se você executar o método de envio de e-mail em um loop, permitindo que os dados sejam redefinidos entre os ciclos.

foreach ($list as $name => $address)
{
        $this->email->clear();

        $this->email->to($address);
        $this->email->from('your@example.com');
        $this->email->subject('Here is your info '.$name);
        $this->email->message('Hi '.$name.' Here is the info you requested.');
        $this->email->send();
}

Se você definir o parâmetro como TRUE, todos os anexos serão apagados também:

$this->email->clear(TRUE);
send([$auto_clear = TRUE])¶
Parâmetros:
  • $auto_clear (bool) – Se deseja limpar dados de mensagens automaticamente
Retorna:

TRUE em caso de sucesso, FALSE em falha
Tipo de retorno:

booleano

O método de envio de e-mail. Retorna booleano TRUE ou FALSE com base no sucesso ou na falha, permitindo que ele seja usado condicionalmente:

if ( ! $this->email->send())
{
        // Geração do erro
}

Este método apagará automaticamente todos os parâmetros se a solicitação for bem-sucedida. Para interromper esse comportamento FALSE

if ($this->email->send(FALSE))
{
        // Os parâmetros não serão apagados
}

Nota

Para usar o método print_debugger(), você precisa evitar limpar os parâmetros de email.

attach($filename[, $disposition = ''[, $newname = NULL[, $mime = '']]])¶
Parâmetros:
  • $filename (string) – Nome do arquivo
  • $disposition (string) 'Disposição' do anexo. A maioria dos clientes de e-mail toma sua própria decisão, independentemente da especificação MIME usada aqui. https://www.iana.org/assignments/cont-disp/cont-disp.xhtml
  • $newname (string) – Nome de arquivo personalizado para usar no e-mail
  • $mime (string) – Tipo MIME a ser usado (útil para dados em buffer)
Retorna:

Instância do CI_Email (método de encadeamento)
Tipo de retorno:

CI_Email

Permite enviar um anexo. Coloque o caminho/nome do arquivo no primeiro parâmetro. Para anexos múltiplos use o método várias vezes. Por exemplo:

$this->email->attach('/path/to/photo1.jpg');
$this->email->attach('/path/to/photo2.jpg');
$this->email->attach('/path/to/photo3.jpg');

Para usar a disposição padrão (anexo), deixe o segundo parâmetro em branco, caso contrário, use uma disposição personalizada:

$this->email->attach('image.jpg', 'inline');

Você também pode usar uma URL:

$this->email->attach('http://example.com/filename.pdf');

Se você quiser usar um nome de arquivo personalizado, você pode usar o terceiro parâmetro:

$this->email->attach('filename.pdf', 'attachment', 'report.pdf');

Se você precisar usar uma cadeia de buffer em vez de um arquivo real - físico, você pode usar o primeiro parâmetro como buffer, o terceiro parâmetro como nome do arquivo e o quarto parâmetro como mime-type:

$this->email->attach($buffer, 'attachment', 'report.pdf', 'application/pdf');
attachment_cid($filename)¶
Parâmetros:
  • $filename (string) – Nome do arquivo de anexo existente
Retorna:

Identificação do conteúdo do anexo ou FALSE se não for encontrado
Tipo de retorno:

string

Define e retorna o ID de conteúdo de um anexo, o que permite que você incorpore um anexo inline (imagem) no HTML. O primeiro parâmetro deve ser o nome de arquivo já anexado.

$filename = '/img/photo1.jpg';
$this->email->attach($filename);
foreach ($list as $address)
{
        $this->email->to($address);
        $cid = $this->email->attachment_cid($filename);
        $this->email->message('<img src="cid:'. $cid .'" alt="photo1" />');
        $this->email->send();
}

Nota

O ID de conteúdo para cada e-mail deve ser recriado para que seja exclusivo.

print_debugger([$include = array('headers', 'subject', 'body')])¶
Parâmetros:
  • $include (array) – Quais partes da mensagem para imprimir
Retorna:

Dados de depuração formatados
Tipo de retorno:

string

Retorna uma string contendo as mensagens do servidor, os cabeçalhos de e-mail e a mensagem de e-mail. Útil para depuração.

Você pode opcionalmente especificar quais partes da mensagem devem ser impressas. As opções válidas são: headers, subject, body.

Exemplo:

// Você precisa passar FALSO ao enviar para obter os dados de e-mail para
// não ser limpo - se isso acontecer, print_debugger () teria
// nada para a saída.
$this->email->send(FALSE);

// Apenas imprimirá os cabeçalhos de e-mail, excluindo o assunto e o corpo da mensagem
$this->email->print_debugger(array('headers'));

Nota

Por padrão, todos os dados brutos serão impressos.
