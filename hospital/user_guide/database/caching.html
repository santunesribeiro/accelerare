Classe de Caching de Banco de dados¶

A Classe de Caching de Banco de Dados permite que você faça cache das suas consultas em arquivos de textos para reduzir a carga do banco de dados.

Importante

Esta classe é inicializada automaticamente pelo driver de banco de dados quando o caching está ativado. NÃO carregue esta classe manualmente.

Importante

Nem todos as funções de resultados de consultas estão disponíveis quando você usa caching. Por favor, leia esta página cuidadosamente.

Habilitando Caching¶

O Cache é ativado em 3 etapas:

  • Crie um diretório gravável no seu servidor onde os arquivos de cache possam ser armazenados.
  • Defina o caminho da sua pasta de cache no arquivo application/config/database.php
  • Habilite o recurso de caching, de forma global definindo esta preferência no arquivo application/config/database.php, ou manualmente como descrito abaixo:

Uma vez ativada, o caching acontecerá automaticamente sempre que uma página carregada possua consultas de banco de dados.

Como o Caching funciona?¶

O sistema de caching de consultas do Codeigniter acontece dinamicamente quando suas páginas são visualizadas. Quando o caching está habilitado, a primeira vez que a página web é carregada, o objeto resultado da consulta será serializado e armazenado em um arquivo de texto em seu servidor. A próxima vez em que a página é carregada o arquivo de cache será usado em vez de acessar seu banco de dados. O uso de banco de dados pode efetivamente ser reduzido a zero para todas as páginas que foram armazenadas em cache.

Somente consultas do tipo leitura (SELECT) podem ser armazenadas em cache, uma vez que estes são os únicos tipos de consultas que produzem resultado. As consultas do tipo escrita (INSERT, UPDATE, eta.), uma vez que não produzem resultados, não serão armazenadas em cache pelo sistema.

Arquivos de cache NÃO expiram. Qualquer consulta que foi armazenada em cache irá permanecer desta forma até que o arquivo de cache seja deletado. O sistema de caching permite você limpar caches associados com páginas individuais, ou você pode deletar um coleção inteira de arquivos de cache. Geralmente você desejará usar as funções de limpeza descritas abaixo para deletar arquivos de cache depois de certos eventos serem alterados, como quando você adicionou novas informações para o seu banco de dados.

O Cache vai melhorar a performance do seu site?¶

Conseguir um ganho de desempenho como resultado de cache depende de muitos fatores. Se você tem um banco de dados altamente otimizado sob muito pouca carga, você provavelmente não verá um aumento de desempenho. Se o seu banco de dados está em uso pesado, você provavelmente vai ver uma melhor resposta, assumindo que o seu sistema de arquivos não é excessivamente sobrecarregado. Lembre-se que o cache simplesmente muda a forma como a sua informação é recuperada, divergindo-o de ser uma operação de banco de dados para um sistema de arquivos.

Em alguns ambientes de servidores em cluster, por exemplo, o armazenamento em cache pode ser prejudicial, uma vez as operações do sistema de arquivos são são muito intensos. Em servidores em ambientes compartilhados únicos, caching provavelmente será benéfico. Infelizmente não há uma resposta única para a questão de saber se você deve armazenar em cache o seu banco de dados. Isto realmente depende de sua situação.

Como os arquivos de Cache são armazenados?¶

O Codeigniter coloca o resultado de casa consulta dentro de um arquivo de cache próprio. Conjuntos de arquivos de cache são ainda organizados em sub-pastas correspondentes às suas funções de controlador. Para ser mais preciso, as sub-pastas são nomeadas de forma idêntica para os dois primeiros segmentos de sua URI (nome da classe controladora e nome da função)

Vamos dizer por exemplo que você tenha um controlador chamado blog com a função chamada comentarios que contenha 3 consultas. O sistema de cache irá criar uma pasta de cache chamada blog+comentarios e dentro irá criar 3 arquivos de cache.

Se você usa consultas dinâmicas que se alteram baseada nas informações em sua URI (quando usamos paginação, por exemplo), cada instância da consulta irá produzir seu próprio arquivo de cache. É possível, portanto, acabar muitas vezes com mais arquivos de cache do que você tem consultas.

Gerenciando seus arquivos de cache¶

Desde que os arquivos de cache não expirem, você vai precisar construir rotinas de exclusão na sua aplicação. Por exemplo, vamos dizer que você tem um blog que permite que os usuários comentem. Sempre que um novo comentário é submetido você vai querer excluir os arquivos de cache associados com a função controle que serve seus comentários. Você irá encontrar duas funções de deleção descritas abaixo que lhe ajudará a esclarecer estas informações.

Nem todas as funções de Banco de Dados funcionam com Caching¶

Por fim, precisamos salientar que o objeto resultado que é inserido em cache é uma versão simplificada do resultado do objeto inteiro. Por esta razão, algumas das funções de resultados de consultas não estão disponíveis para uso.

A seguinte função NÃO ESTÁ disponível ao usar um resultado em cache do objeto:

  • num_fields()
  • field_names()
  • field_data()
  • free_result()

E também, os dois recurso de banco de dado (result_id e conn_id) não estão disponível ao armazenar em cache, já que os recursos de resultado referem-se apenas ao tempo de execução das operações.

Funções de Referência¶

$this->db->cache_on() / $this->db->cache_off()¶

Caching pode ser habilitado/desabilitado manualmente. Isso pode ser útil se você deseja manter certas consultas sendo colocadas em cache. Por exemplo:

// Ativa o cache
$this->db->cache_on();
$query = $this->db->query("SELECT * FROM mytable");

// Desativa o cache para apenas esta consulta
$this->db->cache_off();
$query = $this->db->query("SELECT * FROM members WHERE member_id = '$current_user'");

// Ativa o cache novamente
$this->db->cache_on();
$query = $this->db->query("SELECT * FROM another_table");

$this->db->cache_delete()¶

Exclui os arquivos de cache associado a uma determinada página. Isso é útil se você precisa limpar o cache depois que você atualizar seu banco de dados.

O sistema de cache salva seus arquivos de cache para pastas que correspondem a o URI da página que você está visualizando. Por exemplo, se você estiver visualizando uma página em example.com/index.php/blog/comentarios, o sistema de cache vai colocar todos os arquivos de cache associados em uma pasta chamada do blog + comentários. Para excluir esses arquivos de cache particulares que você vai usar:

$this->db->cache_delete('blog', 'comentarios');

Se você não usa nenhum parâmetro, a atual URI será usada quando determinar o que deve ser limpo.

$this->db->cache_delete_all()¶

Limpa todos os arquivos de cache existentes. Exemplo: 

$this->db->cache_delete_all();
