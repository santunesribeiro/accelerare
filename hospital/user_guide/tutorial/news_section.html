Seção de notícias¶

Na última parte, nós passamos por um conceito básico do framework, escrevendo uma classe que incui páginas estáticas. Nós limpamos o URI adicionando regras de roteamento personalizado. Agora é hora de introduzir um conteúdo dinâmico e começar a usar um banco de dados.

Configurando seu model¶

Ao invés de escrever operações de banco de dados diretamente no controlador, queries deverão ser colocadas em um model, para que eles possam ser facilmente reutilizados mais tarde. Os model são o lugar onde você recuperar, insere e atualizar as informações em seu banco de dados ou outros armazenamentos de dados. Eles representam seus dados.

Abra o diretório application/models/ e dentro dele crie um novo arquivo chamado News_model.php e adicione o seguinte código. Certifique-se de que tenha configurado seu banco de dados corretamente, conforme descrito aqui.

<?php
class News_model extends CI_Model {

        public function __construct()
        {
                $this->load->database();
        }
}

Este código é semelhante ao código do controller que foi usado anteriormente. Ele cria um novo model extendendo de CI_Model e carrega a biblioteca de banco de dados. Isto irá fazer com que a classe de banco de dados fique disponível através do objeto $this->db.

Antes de consultar o banco de dados, um esquema do banco de dados tem que ser criado. Conecte seu banco de dados e execute o comando SQL abaixo (MySQL). Também adicione alguns registros na base.

CREATE TABLE news (
        id int(11) NOT NULL AUTO_INCREMENT,
        title varchar(128) NOT NULL,
        slug varchar(128) NOT NULL,
        text text NOT NULL,
        PRIMARY KEY (id),
        KEY slug (slug)
);

Agora que o banco de dados e um model foram criados, você irá precisar de um método para obetr todos os nossos posts em nosso banco de dados. Para fazer isso, é usado a Classe de construção de consulta (Query Builder) — que é a camada de abstração do banco de dados que já vem incluído com o CodeIgniter. Isso permite que escreva suas ‘consultas’ uma vez e fazer com que funcione em todos os sistemas de bancos de dados suportados. Adicione o seguinte código em seu model.

public function get_news($slug = FALSE)
{
        if ($slug === FALSE)
        {
                $query = $this->db->get('news');
                return $query->result_array();
        }

        $query = $this->db->get_where('news', array('slug' => $slug));
        return $query->row_array();
}

Com esse código você pode executar duas consultas diferentes. Você pode obter todos os registros de notícias, ou obter uma notícia por por seu slug. Você deve ter notado que a variável $slug não foi limpa antes de executar a consulta; Query Builder faz isso para você.

Exibir as notícias¶

Agora que as consultas foram escritas, o model deve ser ligado para as views que irão exibir os items das notícias para o usuário. Isso poderia ser feito no nosso controller Pages criadas anteriormente, mas por uma questão de clareza, um novo controller News está definido. Crie o novo controller em application/controllers/News.php.

<?php
class News extends CI_Controller {

        public function __construct()
        {
                parent::__construct();
                $this->load->model('news_model');
                $this->load->helper('url_helper');
        }

        public function index()
        {
                $data['news'] = $this->news_model->get_news();
        }

        public function view($slug = NULL)
        {
                $data['news_item'] = $this->news_model->get_news($slug);
        }
}

Olhando para o código, você pode ver alguma semelhança com os arquivos que criamos anteriormente. Primeiro o método __construct(): este chama o construtor de sua classe pai (CI_Controller) e carrega o model, para que ele possa ser usado em todos os outros métodos neste controller. Também carrega uma coleção de funções Auxiliar de URL, porque usaremos uma delas em uma view mias tarde.

Em seguida, existem dois métodos para exibir todas as notícias e um para um item de notícia específico. Você pode ver que a variável $slug é passada para o método do model no segundo método. O model esta usando este slug para identificar o item de notícia a ser retornado.

Agora os dados são recuperados pelo controller através de nosso model, mas nada é exibido ainda. A próxima coisa a fazer é passar esses dados para as views.

public function index()
{
        $data['news'] = $this->news_model->get_news();
        $data['title'] = 'News archive';

        $this->load->view('templates/header', $data);
        $this->load->view('news/index', $data);
        $this->load->view('templates/footer');
}

O código acima recebe todos os registros de notícias do model e atribui a uma variável. O valor para o título também é atribuído ao elemento $data['title'] e todos os dados são passados para a view. Você agora precisa criar uma view para renderizar os itens das notícias. Crie application/views/news/index.php e adicione o próximo trecho ao código.

<h2><?php echo $title; ?></h2>

<?php foreach ($news as $news_item): ?>

        <h3><?php echo $news_item['title']; ?></h3>
        <div class="main">
                <?php echo $news_item['text']; ?>
        </div>
        <p><a href="<?php echo site_url('news/'.$news_item['slug']); ?>">View article</a></p>

<?php endforeach; ?>

Aqui, é realizado um laço de repetição para exibir cada item de notícia para o usuário. Você pode ver que nós escrevemos nosso template em PHP misturado com HTML. Se você preferir usar uma linguagem de template, você pode usar a classe Template Parser do CodeIgniter ou um parser de terceiro.

Uma visão geral da página de notícias está completo agora, mas a página para exibir um item de notícia individualmente ainda está ausente. O model criado anteriormente é feito de tal forma que pode ser facilmente utilizado para esta funcionalidade. Você só precisa adicionar um código no controller e criar uma nova view. Vamos voltar para o controller News e atualizar o método view() com o seguinte:

public function view($slug = NULL)
{
        $data['news_item'] = $this->news_model->get_news($slug);

        if (empty($data['news_item']))
        {
                show_404();
        }

        $data['title'] = $data['news_item']['title'];

        $this->load->view('templates/header', $data);
        $this->load->view('news/view', $data);
        $this->load->view('templates/footer');
}

Em vez de chamar o método get_news() sem um parâmetro, a variável $slug é passada, então irá retornar um item de notícia específico. As únicas coisas que restam a faer é criar uma view correspondente em application/views/news/view.php. Coloque o seguinte código neste arquivo.

<?php
echo '<h2>'.$news_item['title'].'</h2>';
echo $news_item['text'];

Roteamento¶

Devido a regra de roteamento criado anteriomento, você precisa de uma rota extra para exibir o controller que acabou de fazer. Modifique seu arquivo de rota (application/config/routes.php) para que fique como segue. Isso garante que as requisições cheguem ao controller News em vez de ir diretamente para o controller Pages. A primeira linha roteia URI’s com um slug para o método view() no controller News.

$route['news/(:any)'] = 'news/view/$1';
$route['news'] = 'news';
$route['(:any)'] = 'pages/view/$1';
$route['default_controller'] = 'pages/view';

Aponte seu navegador para o documento root, seguido por index.php/news e veja sua página de notícias.
