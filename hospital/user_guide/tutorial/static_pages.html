Páginas estáticas¶

Nota: Este tutorial assume que você tenha baixado o CodeIgniter e instalado o framework em seu ambiente de desenvolvimento.

A primeira coisa que você vai fazer é criar um controller para lidar com páginas estáticas. Um controller é simplesmente uma classe que ajuda o trabalho delegado. É a cola da sua aplicação web.

Por exemplo, quando uma chamada é feita a:

http://example.com/noticias/recentes/10

Podemos imaginar que existe um controller chamado "noticias". O método que está sendo chamado em noticias é "recentes". O trabalho desse método em noticia é de pegar 10 itens de notícias e renderiza-los na página. Muitas vezes, no MVC, você irá ver padrões de URL que correnpondam com:

http://example.com/[controller-class]/[controller-method]/[argumentos]

Como esquemas de URL se tornam mais complexas, isso pode mudar. Mas, por agora, isso é tudo que precisamos saber.

Crie um arquivo em application/controllers/Pages.php com o seguinte código.

<?php
class Pages extends CI_Controller {

        public function view($page = 'home')
        {
        }
}

Você criou uma classe chamada Pages, com o método view que aceita um argumento chamado $page. A classe Pages extende da classe CI_Controller. Isso significa que a nova classe Pages pode acessar os métodos e variáveis definidas na classe CI_Controller (system/core/Controller.php).

O controller é o que vai ser o centro de toda requisição da sua aplicação web. Em debates mais técnicos do CodeIgniter, pode ser referido como o super objeto. Como qualquer classe php, você se refere a ele dentro dos seus controllers como $this. Referindo-se a $this é como você vai carregar bibliotecas, views, e geralmente comandar o framework.

Agora que você criou seu primeiro método, é hora de fazer um template básico da página. Nós iremos criar duas "views" (template de página) que irão atuar como sua página de rodapé e cabeçalho.

Crie o cabeçalho em application/views/templates/header.php e adicione o seguinte código:

<html>
        <head>
                <title>CodeIgniter Tutorial</title>
        </head>
        <body>

                <h1><?php echo $title; ?></h1>

O cabeçalho contém o código HTML básico que você deseja mostrar antes de de carregar a view principal, junto com um título. A saída irá ser também a variável $title, que iremos definir depois no controller. Agora, crie um rodapé em application/views/templates/footer.php incluindo o seguinte código:

                <em>&copy; 2015</em>
        </body>
</html>

Adicionando lógica ao controller¶

No início, você criou um controller com um método view(). O método aceita um parâmetro, que é o nome da página a ser carregada. A página estática de template estará disponível no diretório application/views/pages/.

Neste diretório, crie dois arquivos denominados home.php e about.php. Dentro desses arquivos, digite algum texto - qualquer coisa que queira - e salve ele. Se quiser ser particularmente não original, tente “Olá Mundo!”.

Para carregar as páginas, você tem que verificar se a página solicitada realmente existe:

public function view($page = 'home')
{
        if ( ! file_exists(APPPATH.'views/pages/'.$page.'.php'))
        {
                // Ops, não temos uma página para isso!
                show_404();
        }

        $data['title'] = ucfirst($page); // Coloca a primeira letra em maiúscula

        $this->load->view('templates/header', $data);
        $this->load->view('pages/'.$page, $data);
        $this->load->view('templates/footer', $data);
}

Agora, quando a página não existir, ele é carregado, incluindo o cabeçalho e o rodapé, e exibido ao usuário. Se a página não existir, o erro é exibido "404 Página não encontrada".

A primeira linha desse método verifica se a página realmente existe. A função nativa do PHP file_exists() é usada para verificar se o arquivo esta onde é esperado para estar. A função show_404() esta embutida no CodeIgniter que apresenta a página de erro padrão.

No template header, a variável $title foi usada para personalizar o título da página. O valor do título é definido neste método, mas ao invés de atribuir o valor para uma variável, é atribuído ao elemento title no array $data.

A última coisa que tem que ser feito é carragear as views na ordem em que devem ser exibidas. O segundo parâmetro no método view() é usado para passar valores para a view. Cada valor no array $data é atribuído a uma variável com o nome de sua chave. Portanto, o valor de $data['title'] no controller é equivalente a $title na view.

Roteamento¶

O controller agora está funcionando! Aponte no seu navegador para [sua-url-do-site]index.php/pages/view para ver sua página. Quando visitar index.php/pages/view/about você irá ver a página about, mais uma vez, incluindo o cabeçalho e o rodapé.

Usando regras de roteamento personalizado, você tem o poder de mapear qualquer URI para qualquer controller e método, e ficar livre da convenção normal: http://example.com/[controller-class]/[controller-method]/[argumentos]

Vamos fazer isso. Abra o arquivo de roteamento localizado em application/config/routes.php e adicione as duas linhas seguintes. Remova todos outros códigos que estejam definidos no elemento no array $route.

$route['default_controller'] = 'pages/view';
$route['(:any)'] = 'pages/view/$1';

O CodeIgniter lê as regras de roteamento de cima para baixo e encaminha a solicitação para a primeira regra correspondente. Cada regra é uma expressão regular (lado esquerdo) mapeado para um nome do controller e método separados por barras (lado direito). Quando uma solicitação chega, o CodeIgniter olha para a primeira correspondência, e chama o controller e método adequado, possivelmente com argumentos.

Mais informações sobre roteamento podem ser encontrados na documentação Roteamento URI.

Aqui, a segunda regra no array $routes corresponde a qualquer solicitação usando a string de curinga (:any), e passa o parâmetro para o método view() da classe Pages.

Agora visite index.php/about. Será que as páginas são roteadas corretamente para o método view() no controller? Impressionante!
